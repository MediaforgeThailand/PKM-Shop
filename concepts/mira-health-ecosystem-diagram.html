User ⌄

User เริ่มคุยเรื่องสุขภาพ

เริ่มจากปัญหา ความกังวล หรือเป้าหมายด้านสุขภาพ
รายละเอียด
  • User คุยกับผู้ช่วยสุขภาพตั้งแต่เริ่มต้น
  • ข้อมูลที่ user ให้จะเป็น context สำหรับคำแนะนำด้านสุขภาพ
  • ใช้เป็นจุดตั้งต้นของการแนะนำการตรวจสุขภาพ
AI ⌄

AI Health Assistant

ผู้ช่วยด้านสุขภาพที่รู้จัก user มากขึ้นเรื่อย ๆ
รายละเอียด
  • ช่วยเก็บข้อมูลปัญหาของ user ผ่านการสนทนา
  • ให้คำแนะนำด้านการตรวจสุขภาพ
  • ใช้ข้อมูล user และ RAG ทางการแพทย์เพื่อให้คำแนะนำถูกต้องขึ้น
Capture ⌄

Sensitive Data Capture

เจอข้อมูลสำคัญในแชทแล้วเก็บเข้า profile
รายละเอียด
  • ถ้าแชทเจอข้อมูล sensitive หรือข้อมูลใหม่ควรเก็บไว้
  • ตัวอย่างที่ระบุ: ประวัติการรักษา, อายุ, น้ำหนัก, ปัญหาสุขภาพ
  • เป้าหมายคือทำให้ AI เข้าใจ user และแนะนำได้ดีขึ้น
Memory ⌄

User Medical Memory

ข้อมูลการแพทย์และสุขภาพเฉพาะของ user
รายละเอียด
  • เก็บข้อมูลที่ AI ได้จากบทสนทนาและผลตรวจ
  • ทำให้ AI รู้สถานะ user ได้มากขึ้นเมื่อเวลาผ่านไป
  • ข้อมูลสุขภาพต้องมีวันที่กำกับและรู้ว่า update ล่าสุดเมื่อไหร่
RAG ⌄

Medical Knowledge RAG

คลังข้อมูลการแพทย์ที่น่าเชื่อถือสำหรับคำแนะนำ
รายละเอียด
  • ใช้สนับสนุนคำแนะนำจาก AI ให้ถูกต้องขึ้น
  • เป็นแหล่งอ้างอิงสำหรับคำแนะนำใน chat และ dashboard
Guide ⌄

Health Check Guidance

AI แนะนำการตรวจสุขภาพที่เหมาะกับ user
รายละเอียด
  • แนะนำบริการหรือการตรวจสุขภาพจากข้อมูลของ user
  • พา user ไปตรวจสุขภาพกับโรงพยาบาลสมาชิกหรือบริการสุขภาพ
Service ⌄

Member Health Services

โรงพยาบาลสมาชิกและบริการด้านสุขภาพทั้งหมด
รายละเอียด
  • เป็นปลายทางที่ user ไปใช้บริการตรวจสุขภาพ
  • ไม่ใช่ตลาดแพ็คเกจ เป็น service network ที่ AI พา user ไปใช้งาน
Visit ⌄

Booking / Hospital Visit

User ไปตรวจสุขภาพกับบริการหรือโรงพยาบาลสมาชิก
รายละเอียด
  • ระบบพา user ไปสู่ขั้นตอนตรวจสุขภาพจริง
  • หลังตรวจเสร็จ ผลตรวจจะกลับมาเป็นข้อมูลของ user
Scan ⌄

Result Scan / Upload

นำผลตรวจเข้ามาสแกนกับตัว application
รายละเอียด
  • User นำผลตรวจเข้ามาในแอป
  • ผลตรวจถูกสแกนเพื่อใช้เป็นข้อมูลสุขภาพของ user
Data ⌄

Health Data Vault

เก็บผลตรวจเป็น medical data ของ user
รายละเอียด
  • เก็บผลตรวจที่สแกนเข้ามาเป็นข้อมูลของ user คนนั้น
  • เชื่อมกับ medical memory เพื่อให้ AI รู้ข้อมูลของ user มากขึ้น
  • ข้อมูลต้องมีวันที่และสถานะความล่าสุดของข้อมูล
View ⌄

Health Dashboard

Visualization ให้ user เห็นค่าร่างกายและสุขภาพ
รายละเอียด
  • นำค่าต่าง ๆ ที่ตรวจมาทำเป็น visualization
  • ช่วยให้ user เห็นภาพสุขภาพร่างกายของตัวเอง
  • มีคำแนะนำที่อ้างอิงจาก RAG AI ของระบบ
Advice ⌄

Next Health Advice

คำแนะนำต่อยอดจากข้อมูลล่าสุดของ user
รายละเอียด
  • คำแนะนำมาจากข้อมูล user + ผลตรวจ + คลังข้อมูลการแพทย์
  • ส่งกลับไปทำให้ผู้ช่วยสุขภาพแนะนำได้ดีขึ้นในรอบถัดไป
Referral ⌄

Referral Tag

ระบบ tag user จาก link สำหรับ partner
รายละเอียด
  • ใช้ link เพื่อ tag user ที่มาจาก doctor, nurse หรือ content creator
  • ใช้เป็นข้อมูล attribution สำหรับ commission
Ops ⌄

Payment / Settlement

คำนวณ commission และยอดจ่ายที่เกี่ยวข้อง
รายละเอียด
  • จาก concept เดิม: Mira GP 5%
  • Referral commission 5%
  • Hospital / B2B payout 90%
Partner ⌄

Partner / B2B Ops

ฝั่งโรงพยาบาลหรือบริการสุขภาพที่เป็นสมาชิก
รายละเอียด
  • เชื่อมกับบริการสุขภาพหรือโรงพยาบาลสมาชิก
  • รองรับการดำเนินงานฝั่ง B2B จาก concept เดิม
Core loop: ผลตรวจจริง + ข้อมูลจากบทสนทนา กลายเป็น medical memory ของ user และทำให้ AI Health Assistant แนะนำได้ดีขึ้นในรอบถัดไป
Concept หลักที่เปลี่ยน

ไม่วางเป็นตลาดแพ็คเกจ แต่เป็น AI Health Assistant ที่พา user ไปสู่บริการสุขภาพที่เหมาะสม

Moat ของระบบ

medical memory เฉพาะ user จากบทสนทนาและผลตรวจจริง ทำให้ AI รู้จัก user มากขึ้นเรื่อย ๆ

จุดที่ขยายต่อได้

สามารถเพิ่ม node ใหม่ใน canvas ได้ เช่น workflow, admin, consent, data source หรือระบบ service partner เพิ่มเติม
