# V2 Catalog Admin

The prototype hospital product portal has been replaced by the MiraCare v2 admin catalog route at `/admin/catalog`.

Current screens:

- `/admin/catalog`: creates, edits, archives, restores, and uploads images for tenant products in the canonical `products` table.
- `/admin/orders`: reviews submitted orders and updates status through the shared order state machine.
- `/admin/referrers`: manages referrers, commission schemes, and commission entry status.

The old `/hospital-portal`, `/hospital-products`, and `/admin-booking` route files were removed per the technical spec's single `app/admin/` route group requirement.

## Data Model

- `products`: tenant-scoped catalog used by chat, marketplace, partner assisted purchase, and admin.
- `storage.objects` bucket `product-images`: public product images written by tenant admins under their tenant slug path.
- Product rows no longer create `marketplace.product` RAG chunks. The chat orchestrator queries active tenant products directly and sends the current catalog through the published MiraCare prompt variables.

## Admin Flow

```text
Tenant admin signs in
-> opens /admin/catalog
-> creates or edits product fields from the v2 schema
-> optionally uploads product image to product-images
-> catalog_key is generated by the database and shown read-only
-> active products become available to chat, marketplace, and referrer flows
-> archive/restore toggles products.active
```
