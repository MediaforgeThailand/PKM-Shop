# ฝั่งลูกค้า: บทสนทนาบน LINE ตั้งแต่ทักจนของถึงมือ

> **ลูกค้าไม่มีแอป ไม่มีเว็บ ไม่มีหน้าเช็คเอาต์** — ทั้งร้านอยู่ในห้องแชท LINE ห้องเดียว
> พื้นผิวดีไซน์ที่มีจริงคือ **Flex Message** (การ์ดใน LINE) + **รูป QR** + **ข้อความ** เท่านั้น
> **[ห้ามแตะ]** ข้อความทุกบรรทัดในเอกสารนี้คือคอปี้จริงในระบบ ถ้าจะแก้ต้องเสนอเป็นข้อเสนอแยก

---

## 1. ภาพรวมของหนึ่งเทิร์น

ลูกค้าพิมพ์/กดปุ่ม → ระบบเด้ง **loading animation (จุดสามจุด)** ของ LINE ทันที → ตอบกลับ

**[ห้ามแตะ] เพดานสำคัญ: หนึ่งเทิร์นส่งได้สูงสุด 5 ข้อความ** — เกินจากนี้ถูกตัดทิ้งเงียบ ๆ และ **ตัวที่โดนตัดคือ QR กับการ์ดชำระเงินเพราะอยู่ท้ายสุด** ถ้าดีไซน์เสนอให้ส่งการ์ดเพิ่ม ต้องคิดเรื่องนี้ด้วยเสมอ

**สองช่องทาง**:
- **ตอบกลับทันที (reply)** — ตอนลูกค้าคุย/กดปุ่ม
- **แจ้งเตือน (push)** — ตอนสถานะเปลี่ยน (จ่ายผ่าน/แพ็คเสร็จ/ไรเดอร์ออก/ส่งถึง) ลูกค้าไม่ได้ทำอะไร แต่ข้อความเด้งมาเอง

---

## 2. เส้นทางการซื้อ ทีละสเต็ป

### สเต็ป 1 — ทักทาย (ลูกค้าเพิ่มเพื่อน)
- ข้อความต้อนรับ **มาจาก AI ไม่ได้ hardcode** — ดีไซน์ควบคุมไม่ได้ตรง ๆ
- ถ้า AI ล่ม → ข้อความสำรอง: *"ขออภัยค่ะ ระบบตอบอัตโนมัติขัดข้องชั่วคราว ลองพิมพ์อีกครั้ง หรือกดปุ่ม "ดูสินค้า" ได้เลยค่ะ"*

### สเต็ป 2 — ดูสินค้า
| ลูกค้าทำ | ระบบตอบ |
|---|---|
| กดปุ่มจาก rich menu | ข้อความ *"เลือกหมวดที่สนใจได้เลยค่ะ"* + **การ์ดหมวดสินค้า (carousel สูงสุด 10 ใบ)** |
| กดปุ่ม "ดูสินค้า" ในการ์ดหมวด | ข้อความ *"เลือกสินค้าที่สนใจได้เลยค่ะ"* + **การ์ดสินค้า (carousel)** |
| หมวดนั้นของหมด | *"หมวดนี้สินค้าหมดชั่วคราวค่ะ"* (ไม่มีการ์ด) |
| พิมพ์คุยเฉย ๆ | AI ตอบเอง + แนบการ์ดสินค้า **สูงสุด 4 ใบ** |

**การ์ดหมวดสินค้า**: ชื่อหมวด (ตัวหนา) + "{จำนวน} รายการ" + ปุ่ม **"ดูสินค้า"**
**การ์ดสินค้า**: รูปสินค้า (อัตราส่วน 20:13, ครอบเต็ม — **ข้ามรูปถ้าสินค้าไม่มีรูป**) + ชื่อ (ตัวหนา) + คำอธิบาย + ราคา `฿{ราคา}` (ตัวใหญ่ หนา สีแบรนด์) + ปุ่ม **"เลือก"**

> ⚠️ **[ปรับได้ — จุดอ่อนจริง]** ปุ่ม "เลือก" ไม่มีตัวเลือกจำนวน — ลูกค้าเพิ่มจำนวนได้ทางเดียวคือพิมพ์คุยกับ AI

### สเต็ป 3 — เพิ่มลงตะกร้า
- *"เพิ่มลงตะกร้าแล้วค่ะ แจ้งที่อยู่จัดส่งได้เลย (พิมพ์ หรือกดแชร์ตำแหน่งใน LINE)"*
- **ตะกร้า = ออเดอร์สถานะ "รอชำระ"** ไม่มีหน้าตะกร้าแยก

### สเต็ป 4 — ที่อยู่
- ลูกค้าแชร์ตำแหน่งใน LINE หรือพิมพ์ที่อยู่
- สำเร็จ: *"รับที่อยู่แล้วค่ะ เลือกวิธีจัดส่งได้เลย"* + **การ์ดตัวเลือกจัดส่ง**
- ยังไม่มีของในตะกร้า: *"ยังไม่มีสินค้าในตะกร้าค่ะ เลือกสินค้าก่อนนะคะ"*

**ตัวเลือกที่โผล่ขึ้นกับระยะทางจากร้าน** (แอดมินบังคับโซนรายลูกค้าได้ และการบังคับชนะระยะทางเสมอ):
- ในเขต → ไรเดอร์ร้าน / Grab / Kerry
- นอกเขต → Lalamove / Kerry
- ไม่มีพิกัด → ไรเดอร์ / Grab / Kerry (Lalamove ถูกตัดออกเสมอเพราะคิดราคาตามระยะ)

**การ์ดตัวเลือกจัดส่ง**: หัวข้อ "เลือกวิธีจัดส่ง" + บรรทัดละ "{วิธี} — ฿{ค่าส่ง}" + **ปุ่มต่อหนึ่งตัวเลือก** เขียนว่า "{วิธี} ฿{ค่าส่ง}"

### สเต็ป 5 — เลือกวิธีส่ง
- สำเร็จ: *"เลือก {วิธี} ค่าส่ง ฿{ค่าส่ง} — โอนแล้วส่งสลิปได้เลยค่ะ"*
- ตัวเลือกไม่พร้อม: *"วิธีจัดส่งนี้ยังไม่พร้อมค่ะ"* · ไม่มีออเดอร์: *"ยังไม่มีออเดอร์ค่ะ"*
- **จุดนี้คือจุดที่ปลดล็อก QR** — ระบบคำนวณยอดรวมแล้ว

### สเต็ป 6 — QR พร้อมเพย์ + ยอด (**สองข้อความติดกัน**)
1. **รูป QR** — ระบบสร้าง PNG แล้วส่งเป็นรูปในแชท
2. **การ์ดชำระเงิน**: แถบหัวเขียวแบรนด์ตัวอักษรขาว "ชำระเงิน" → เลขออเดอร์ (ตัวหนา) → บรรทัดสินค้า "{ชื่อ} x{จำนวน} — ฿{ยอด}" → "ค่าส่ง ฿{ค่าส่ง}" (ตัวเล็กสีเทา) → **ยอดรวม ฿{ยอด} (ตัวใหญ่สุด หนา สีแบรนด์)** → *"สแกน QR ด้านบนแล้วส่งรูปสลิปในแชทได้เลยค่ะ"*

> ⚠️ **[ปรับได้ — จุดอ่อนจริง]** QR + การ์ดชำระเงินจะถูกส่งซ้ำ **ทุกครั้ง**ที่ลูกค้าพิมพ์อะไรก็ตามขณะตะกร้าพร้อมจ่าย — คุย 5 ประโยค = ได้ QR 5 ชุด ห้องแชทรก

### สเต็ป 7 — ลูกค้าส่งรูปสลิป → ระบบตรวจอัตโนมัติ
ระบบตรวจสลิปเองทันที ไม่ผ่าน AI เลย แล้วตอบ **ข้อความล้วน** ตามผล:

| ผล | ข้อความจริง |
|---|---|
| ผ่าน (ค่าสินค้า) | *"ชำระเงินสำเร็จค่ะ ✅ ออเดอร์ {เลข} กำลังเข้าคิวจัดส่ง เดี๋ยวระบบแจ้งรอบให้ทันทีนะคะ"* |
| ผ่าน (ค่าส่งใหม่) | *"ได้รับค่าส่งแล้วค่ะ ✅ ระบบสร้างออเดอร์จัดส่งใหม่ {เลข} ให้เรียบร้อย เดี๋ยวแจ้งรอบจัดส่งอีกครั้งนะคะ"* |
| จ่ายไปแล้ว | *"ออเดอร์ {เลข} ชำระเงินเรียบร้อยแล้วค่ะ ไม่ต้องส่งสลิปซ้ำนะคะ"* |
| ธนาคารช้า | *"ธนาคารต้นทางกำลังประมวลผลค่ะ รอประมาณ {นาที} นาที แล้วส่งสลิปเดิมอีกครั้งได้เลยนะคะ"* |
| ธนาคารล่ม | *"ระบบธนาคารขัดข้องชั่วคราวค่ะ อีกสักครู่รบกวนส่งสลิปเดิมอีกครั้งนะคะ"* |
| สลิปซ้ำ | *"สลิปนี้เคยถูกใช้ยืนยันแล้วค่ะ ถ้าโอนใหม่รบกวนส่งสลิปของรายการล่าสุดนะคะ"* |
| ยอดไม่ตรง | *"ยอดในสลิปไม่ตรงกับยอดที่ต้องชำระ ({ยอด} บาท) ค่ะ รบกวนตรวจสอบแล้วส่งสลิปที่ยอดถูกต้องนะคะ"* |
| อ่านสลิปไม่ออก | *"อ่านข้อมูลจากสลิปไม่ได้ค่ะ รบกวนส่งรูปสลิปที่เห็น QR ชัด ๆ อีกครั้งนะคะ"* |
| เข้าคิวตรวจมือ | *"รับสลิปแล้วค่ะ ส่งให้เจ้าหน้าที่ตรวจสอบ จะยืนยันให้เร็วที่สุดนะคะ 🙏"* |
| ระบบตรวจล่ม | *"รับสลิปแล้วค่ะ ระบบตรวจสอบอัตโนมัติขัดข้อง ส่งให้เจ้าหน้าที่ตรวจสอบแล้ว จะยืนยันให้เร็วที่สุดนะคะ 🙏"* |
| ส่งสลิปถี่เกิน | *"ส่งสลิปถี่เกินไปค่ะ รอสักครู่แล้วส่งอีกครั้งนะคะ 🙏"* |

### สเต็ป 8 — แจ้งเตือนหลังจากนี้ (ลูกค้าไม่ได้ทำอะไร ข้อความเด้งเอง)

| จังหวะ | ข้อความจริง |
|---|---|
| จ่ายผ่าน (มีรอบ) | *"ชำระเงินสำเร็จ ✅ ออเดอร์ {เลข} เข้ารอบจัดส่ง {เวลา} น. เตรียมรับของได้เลยค่ะ"* |
| จ่ายผ่าน (ไม่มีรอบ) | *"ชำระเงินสำเร็จ ✅ ออเดอร์ {เลข} กำลังเตรียมจัดส่งค่ะ"* |
| แพ็คเสร็จ | *"แพ็คสินค้าเรียบร้อย 📦 ออเดอร์ {เลข} กำลังเข้าสู่ขั้นตอนจัดส่งค่ะ"* |
| ไรเดอร์รับรอบ | *"ออเดอร์ {เลข} อยู่ในรอบ {เวลา} น. ไรเดอร์กำลังจะออกไปส่ง เตรียมรับของค่ะ 🛵"* |
| **ถึงคิวคุณแล้ว** | *"ไรเดอร์กำลังจัดส่งไปหาคุณแล้ว 🛵 ออเดอร์ {เลข}"* |
| ส่งถึง | *"จัดส่งสำเร็จ 🎉 ขอบคุณที่อุดหนุนค่ะ ออเดอร์ {เลข}"* (แนบรูปหลักฐานมาก่อนข้อความ) |
| ตีกลับ | *"ออเดอร์ {เลข} ตีกลับ เนื่องจาก: {เหตุผล}"* + *"กรุณาชำระค่าส่งใหม่ {ค่าส่ง} บาท เพื่อจัดส่งอีกครั้งค่ะ"* |
| ส่ง Kerry แล้ว | *"ส่งมอบพัสดุให้ Kerry แล้วค่ะ เลขพัสดุ {เลข} (ออเดอร์ {เลข})"* |
| สลิปถูกปฏิเสธ | *"สลิปของออเดอร์ {เลข} ไม่ผ่านการตรวจสอบค่ะ ({เหตุผล})"* + *"รบกวนตรวจสอบยอดโอนแล้วส่งสลิปใหม่ หรือพิมพ์คุยกับแอดมินได้เลยค่ะ"* |

> ⚠️ **[ปรับได้ — จุดอ่อนจริง]** ตอนจ่ายผ่าน ลูกค้าได้ **2 ข้อความความหมายซ้ำกัน** ติด ๆ กัน (ข้อความตอบสลิป + แจ้งเตือน) ต่างกันแค่อันหลังบอกเวลารอบ

## 3. เช็คสถานะออเดอร์
ลูกค้าถาม "ของถึงไหน" → **การ์ดสถานะ**: หัวข้อ "สถานะออเดอร์ล่าสุด" + รายการสูงสุด 5 ออเดอร์ แต่ละอันมีเลขออเดอร์ (ตัวหนา) แล้ว "{สถานะไทย} · ฿{ยอด}" — **การ์ดนี้อ่านอย่างเดียว ไม่มีปุ่มเลย**

> ⚠️ **[ปรับได้ — จุดอ่อนชัดเจน]** ลูกค้าเห็นสถานะแต่กดอะไรต่อไม่ได้เลย — นี่คือการ์ดที่ควรมีปุ่มมากที่สุด

## 4. ขอคุยกับคน (handoff)

**ทริกเกอร์ 3 ทาง**: (1) ลูกค้าพิมพ์วลีในลิสต์ 13 วลี — *ขอคุยกับคน, คุยกับคนจริง, ขอคุยกับแอดมิน, คุยกับแอดมิน, ขอคุยกับพนักงาน, ติดต่อพนักงาน, ติดต่อแอดมิน, ติดต่อเจ้าหน้าที่, ขอสายเจ้าหน้าที่, ขอเจ้าหน้าที่, ร้องเรียน, ไม่อยากคุยกับบอท, ไม่คุยกับบอท, ขอคนตอบ* · (2) มีคำว่า "ร้องเรียน" → ติดธงร้องเรียน · (3) AI เองตัดสินใจส่งต่อ

**ลูกค้าเห็น**: *"รับเรื่องแล้วค่ะ กำลังส่งต่อให้เจ้าหน้าที่มาดูแลต่อ รอสักครู่นะคะ 🙏"*

**หลังจากนั้น AI เงียบสนิท** — ลูกค้าพิมพ์อะไรมาก็ไม่มีใครตอบจนกว่าแอดมินจะเข้ามาในคอนโซล **รวมถึงรูปด้วย** (ส่งสลิปมาตอนนี้ระบบไม่ตรวจ เก็บให้แอดมินดูเฉย ๆ)

**แอดมินได้แจ้งเตือน**: *"💬 ลูกค้าขอคุยกับแอดมิน ({เหตุผล}) — เปิดหน้า "แชทลูกค้า" เพื่อรับเรื่อง AI หยุดตอบห้องนี้แล้ว"*

> ⚠️ **[ปรับได้ — จุดอ่อนใหญ่]** ลูกค้าไม่มีทางรู้เลยว่ารออยู่นานแค่ไหน ไม่มีสัญญาณว่า "แอดมินเห็นแล้ว" ไม่มีเวลาทำการ — เงียบคือเงียบ

## 5. ตีกลับ → จ่ายค่าส่งใหม่ → ส่งรอบสอง

1. ไรเดอร์กดตีกลับ + กรอกเหตุผล → ลูกค้าได้ข้อความตีกลับ + ยอดค่าส่งใหม่
2. ออเดอร์เดิมค้างที่ "รอชำระค่าส่งใหม่" **ตลอดไป** — ไม่มีทางกลับ
3. ลูกค้าส่งสลิปค่าส่ง → ระบบสร้าง **ออเดอร์ใหม่** ผูกกับออเดอร์เดิม แล้วเดินเข้าคิวแพ็ค/รอบส่งตามปกติ

> **[ห้ามแตะ]** ค่าส่งรอบใหม่ต้องเท่ากับ **ค่าส่ง** ไม่ใช่ยอดรวมทั้งบิล (ของยังอยู่ที่ร้าน ลูกค้าจ่ายค่าสินค้าไปแล้ว)

## 6. ช่องว่างที่พบในโค้ด (ข้อเท็จจริง ไม่ใช่ข้อเสนอ)

1. **ไม่มี quick reply เลยแม้แต่จุดเดียวทั้งระบบ** — ทางเลือกทั้งหมดเป็นปุ่มในการ์ด Flex ซึ่งค้างอยู่ในประวัติแชทและ**กดซ้ำได้เรื่อย ๆ** (ต่างจาก quick reply ที่หายไปเมื่อตอบแล้ว) → ลูกค้าเลื่อนขึ้นไปกดปุ่มเก่าได้ตลอด
2. **ข้อความสำรองตอน AI ล่มพูดถึงปุ่ม "ดูสินค้า"** ที่ต้องมาจาก rich menu — ถ้าไม่มี rich menu ลูกค้าจะงงว่าปุ่มอยู่ไหน
3. **บางปุ่มตอบกลับเป็นข้อความว่าง** — ส่งแค่ QR + การ์ด ไม่มีข้อความนำ
4. **ไม่มี rich menu ในโค้ด** — ต้องตั้งค่าใน LINE OA Manager แยก **นี่คือพื้นผิวดีไซน์ที่ยังว่างเปล่าทั้งหมด**

## 7. งานดีไซน์ฝั่ง LINE ที่มีค่าที่สุด

1. **Rich menu** — ยังไม่มีเลย ทั้งที่ระบบอ้างถึง ถือเป็นพื้นที่ว่างที่ใหญ่ที่สุด
2. **การ์ดสถานะออเดอร์** — ปัจจุบันอ่านอย่างเดียว ควรมีทางไปต่อ
3. **จังหวะการส่ง QR ซ้ำ** — ต้องแก้ที่ประสบการณ์ ไม่ใช่แค่ความสวย
4. **ข้อความซ้ำตอนจ่ายสำเร็จ** — ยุบเหลือหนึ่ง
5. **ประสบการณ์ตอนรอแอดมิน** — ลูกค้าถูกทิ้งให้เงียบ
6. **การ์ดสินค้าไม่มีตัวเลือกจำนวน** — ซื้อ 2 ชิ้นต้องพิมพ์เอา

> ทุกข้อข้างบนเป็น **[ปรับได้]** — แต่ต้องอยู่ในกรอบ: **สูงสุด 5 ข้อความต่อเทิร์น** และ **ทุกอย่างต้องทำงานในกล่องแชท LINE เท่านั้น**
