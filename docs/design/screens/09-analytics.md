# สถิติ (`/admin/analytics`)

> แอดมิน (เจ้าของร้าน/ผู้ดูแล) เปิดดูภาพรวมยอดขายในช่วงวันที่ที่เลือก และค้นประวัติออเดอร์รายตัวด้วยเลขออเดอร์ — เป็นหน้าอ่านอย่างเดียว ไม่มีการแก้ไขข้อมูลใดๆ

---

## 1. บริบทการใช้งาน

**ใครใช้:** แอดมินเท่านั้น (role `admin`). role อื่นทั้งหมด — `stock`, `packer`, `rider`, `staff` — เข้าหน้านี้ไม่ได้เลย ระบบเด้งออกทันทีโดยไม่มีข้อความแจ้ง และเมนู "สถิติ" (ไอคอน 📈) ก็ไม่ถูกแสดงใน navigation ให้เห็นตั้งแต่แรก

**ใช้บนอุปกรณ์อะไร:** ทั้งมือถือและเดสก์ท็อป แต่พฤติกรรมจริงคือ **นั่งดู** — เปิดตอนเช้าดูยอดเมื่อวาน หรือสิ้นเดือนดูยอดรวมเดือน ไม่ใช่หน้าที่ใช้กลางแดดถือของมือเดียวแบบ `/rider` หรือ `/packer`

**สถานการณ์จริง:**
- เช้า/เย็น: เปิดเช็คว่าวันนี้ขายได้เท่าไหร่ กี่บ้าน
- สิ้นเดือน: เลือกช่วงทั้งเดือน ดูยอดรวมและวันขายดีสุด
- ลูกค้าโทรมาถาม "ของถึงไหนแล้ว": แอดมินเลื่อนลงล่างสุด พิมพ์เลขออเดอร์ แล้วอ่าน timeline สถานะให้ลูกค้าฟังทางโทรศัพท์ — **นี่คือกรณีที่เร่งรีบที่สุดของหน้านี้** และมักเกิดขึ้นบนมือถือ

**ความถี่:** วันละ 1–3 ครั้งสำหรับดูยอด, ส่วนการค้นหาออเดอร์เกิดขึ้นบ่อยกว่า (ทุกครั้งที่ลูกค้าถาม)

**Mobile-first หรือ desktop-first:** **Mobile-first** — ทั้งระบบตั้งพื้นฐานมือถือไว้แล้ว (พื้นที่กดอย่างน้อย 44px, ช่องกรอกฟอนต์ 16px กัน iOS ซูมตอนโฟกัส, เว้น safe-area ทั้งบนล่าง, แถบเมนูล่างสูง 56px และเนื้อหาเผื่อระยะหลบไว้). แต่ต้องเข้าใจว่าหน้านี้เป็น mobile-first ที่ยัง**ทำได้ไม่ดี** — กราฟบนมือถืออ่านค่าตัวเลขไม่ได้เลย (ดูข้อ 8). ดีไซน์ใหม่ต้องคงพื้นฐาน mobile-first ไว้ แต่แก้ให้กราฟใช้งานได้จริงบนมือถือ

**โหมดสี:** light เท่านั้น ทั้งระบบไม่มี dark mode

## 2. โครงหน้า

หน้านี้ **ไม่มีแท็บ** — ทุกอย่างเรียงต่อกันเป็นคอลัมน์เดียวจากบนลงล่าง มีระยะห่างสม่ำเสมอระหว่างบล็อก

**ลำดับบล็อกจากบนลงล่าง:**

1. **หัวข้อหน้า** — ตัวหนังสือ "สถิติ" ขนาดกลาง ตัวหนา ชิดซ้าย. ฝั่งขวา**ว่างเปล่า** ไม่มีปุ่มใดๆ (ไม่มีปุ่ม export, ไม่มีปุ่มรีเฟรช)

2. **การ์ดเลือกช่วงวันที่** — การ์ดพื้นขาว ขอบมน แบ่งเป็น 2 คอลัมน์เท่ากัน **ทุกขนาดจอ** (ไม่มีการปรับตามความกว้าง):
   - ซ้าย: ป้ายกำกับ "ตั้งแต่วันที่" + ช่องเลือกวันที่ (ใช้ปฏิทินของเครื่อง/เบราว์เซอร์ ไม่ใช่ปฏิทินที่ทีมออกแบบเอง)
   - ขวา: ป้ายกำกับ "ถึงวันที่" + ช่องเลือกวันที่แบบเดียวกัน
   - เมื่อช่วงวันที่ผิด: ข้อความสีแดงตัวเล็กแทรกใต้ช่องทั้งสอง กินความกว้างเต็ม 2 คอลัมน์
   - ค่าเริ่มต้น: "ตั้งแต่วันที่" = วันที่ 1 ของเดือนปัจจุบัน, "ถึงวันที่" = วันนี้

3. **แถบเตือนออเดอร์เยอะเกิน** *(แสดงเฉพาะเมื่อมีออเดอร์ตั้งแต่ 1,000 รายการขึ้นไปในช่วงที่เลือก)* — ไม่ใช่การ์ด แต่เป็นแถบบางๆ พื้นเหลืองอ่อน ตัวหนังสือน้ำตาลเข้ม ขอบมนเล็กน้อย มี emoji ⚠️ นำหน้า วางเหนือการ์ดสถิติ

4. **การ์ดสถิติ 3 ใบ** — วางเป็นตาราง:
   - **มือถือ:** ใบที่ 1 กินเต็มความกว้างแถวบน, ใบที่ 2 กับ 3 เรียงคู่กันในแถวล่าง
   - **จอ sm ขึ้นไป:** ทั้ง 3 ใบเรียงในแถวเดียว คอลัมน์ละใบ
   - ใบที่ 1 "ยอดขายรวม" — เน้นพิเศษ: ขอบและพื้นเป็นสีแบรนด์แบบจางๆ ตัวเลขเป็นสีแบรนด์เขียวเข้ม มีคำอธิบายใต้ตัวเลข
   - ใบที่ 2 "จำนวนออเดอร์" — การ์ดขาวธรรมดา มีคำอธิบายใต้ตัวเลข
   - ใบที่ 3 "เฉลี่ย/ออเดอร์" — การ์ดขาวธรรมดา **ไม่มี**คำอธิบาย
   - โครงในแต่ละใบ: ป้ายกำกับตัวเล็กสีเทา → ตัวเลขใหญ่ตัวหนา → คำอธิบายตัวเล็กสีเทาอ่อน (ถ้ามี)

5. **การ์ดกราฟแท่งยอดขายรายวัน** — การ์ดพื้นขาว:
   - หัวการ์ด: หัวข้อ "ยอดขายรายวัน" ชิดซ้าย + ป้ายวันขายดีสุดชิดขวา (ป้ายนี้แสดงเฉพาะเมื่อมีอย่างน้อย 1 วันที่ขายได้). หัวการ์ดขึ้นบรรทัดใหม่ได้เมื่อจอแคบ
   - ตัวกราฟ: แท่งแนวตั้ง 1 แท่งต่อ 1 วัน สูงคงที่ 112px เรียงชิดก้น มีช่องไฟแคบระหว่างแท่ง แต่ละแท่งกว้างสุด 26px หัวแท่งมนบน
   - **ไม่มีแกน Y ไม่มีเส้น gridline ไม่มีตัวเลขบนหัวแท่ง ไม่มี animation ไม่ได้ใช้ chart library ใดๆ**
   - ใต้แท่ง: เลขวันที่ตัวเล็กมาก (10px) และชื่อวันย่อภาษาไทยตัวเล็กกว่านั้นอีก (9px) — **ชื่อวันแสดงเฉพาะเมื่อช่วงที่เลือกยาวไม่เกิน 10 วัน**
   - ถ้าช่วงยาวเกิน 16 วัน กราฟจะกว้างกว่าจอ ต้องปัดซ้าย-ขวา (คำนวณจากจำนวนวัน × 22px)

6. **การ์ดแยกตามวิธีจัดส่ง** *(หายไปทั้งใบเมื่อไม่มีออเดอร์เลย)* — การ์ดพื้นขาว หัวข้อ "แยกตามวิธีจัดส่ง" แล้วเป็นป้าย (badge) พื้นเทาอ่อนเรียงต่อกันแนวนอน ขึ้นบรรทัดใหม่ได้. **ป้ายจะโผล่เฉพาะวิธีส่งที่มีออเดอร์จริงในช่วงนั้น** เรียงตามลำดับตายตัว: ส่งปกติ (ไรเดอร์) → ด่วน (Grab) → นอกเขต (Lalamove) → พัสดุ (Kerry). ไม่มีเปอร์เซ็นต์ ไม่มีกราฟวงกลม ไม่ใช่ตาราง

7. **การ์ดค้นหาออเดอร์** — การ์ดพื้นขาว **แสดงอยู่เสมอ** ไม่ว่าช่วงวันที่จะถูกหรือผิด หรือส่วนสถิติด้านบนจะโหลดพังหรือไม่:
   - หัวข้อ "ค้นหาออเดอร์"
   - ฟอร์มแถวเดียว: ช่องกรอกยืดเต็ม + ปุ่มหลัก (พื้นสีแบรนด์) "ค้นหา" ชิดขวาไม่ยุบ
   - ใต้ฟอร์ม: กล่องผลลัพธ์ (กล่องขอบเทาซ้อนในการ์ด) หรือ empty state หรือ spinner หรือ error — แล้วแต่สถานะ
   - ในกล่องผลลัพธ์เรียงจากบนลงล่าง: แถวเลขออเดอร์ (ฟอนต์ monospace) + ป้ายสถานะ + ป้ายวิธีจัดส่ง → ชื่อผู้รับ · เบอร์ → ที่อยู่ (ตัวเล็กกว่า) → ยอดรวมตัวหนา + รายละเอียดยอดตัวเล็กในวงเล็บ → timeline สถานะ
   - **Timeline:** รายการแนวตั้ง มีเส้นแนวตั้งสีเทาทางซ้าย แต่ละรายการมีจุดวงกลมสีแบรนด์ (ขอบขาว) ทาบบนเส้น + ชื่อสถานะภาษาไทย + บรรทัดวัน/เวลาตัวเล็กสีเทาอ่อน. เรียงเก่า → ใหม่ จากบนลงล่าง

**หมายเหตุโครงสร้าง:** หน้านี้อยู่ใต้ Shell เสมอ — เดสก์ท็อป/แท็บเล็ตมีเมนูด้านบน, มือถือมีแถบแท็บล่างแบบติดหน้าจอ. เนื้อหาหลักจำกัดความกว้างสูงสุดระดับกลาง (max-w-5xl) มีระยะขอบรอบ และเผื่อระยะล่างไว้หลบแถบแท็บบนมือถือ

## 3. ทุก flow ทีละสเต็ป

### 3.1 เข้าหน้าสถิติครั้งแรก (ค่าเริ่มต้น = เดือนนี้)

**ขั้นตอน:**
1. แอดมินกดเมนู "สถิติ" (ไอคอน 📈) จากเมนูด้านบน (เดสก์ท็อป) หรือแถบแท็บล่าง (มือถือ)
2. ระบบตรวจสิทธิ์: ถ้าไม่ใช่ `admin` → เด้งออกไปหน้าแรกของ role นั้นทันที **โดยไม่มีข้อความแจ้งใดๆ** (`stock`→หน้าแคตตาล็อก, `packer`→หน้าแพ็ค, `rider`→หน้าไรเดอร์, อื่นๆ→หน้าสตาฟ)
3. ระบบตั้งช่วงวันที่ให้อัตโนมัติ: "ตั้งแต่วันที่" = วันที่ 1 ของเดือนปัจจุบัน, "ถึงวันที่" = วันนี้
4. ระบบดึงข้อมูลทันทีโดยไม่ต้องกดปุ่มอะไร → แสดงตัวหมุนโหลดพร้อมข้อความ "กำลังโหลด…"
5. โหลดเสร็จ → การ์ดสถิติ 3 ใบ + กราฟ + การ์ดแยกตามวิธีจัดส่ง ปรากฏแทนตัวหมุน
6. การ์ด "ค้นหาออเดอร์" แสดงอยู่แล้วตั้งแต่ขั้นตอนที่ 4 (ไม่รอส่วนสถิติ)

**กฎการตรวจสอบ:**
- ต้องเป็น role `admin` เท่านั้น มิฉะนั้นเด้งออก
- **นับเฉพาะออเดอร์ที่: สถานะ ≠ "ยกเลิก" **และ** สถานะการชำระ = ชำระแล้ว** — ออเดอร์ที่ยังไม่จ่ายหรือถูกยกเลิกจะไม่ปรากฏในตัวเลขและกราฟใดๆ บนหน้านี้เลย (กฎธุรกิจ — ห้ามแตะ)
- ขอบเขตวันใช้ **วันธุรกิจไทย (+07:00)** เสมอ ปลายทางเป็นแบบไม่รวมวันถัดไป — เปิดจากเบราว์เซอร์คนละโซนเวลาก็ไม่ทำให้ยอดเลื่อนวัน
- ดึงได้สูงสุด 5,000 รายการ เรียงตามเวลาสร้างจากเก่าไปใหม่

**ข้อความสำเร็จ:** ไม่มี toast ไม่มีข้อความยืนยันใดๆ — ตัวเลขและกราฟปรากฏแทนตัวหมุนเงียบๆ (**ทั้งหน้าไม่มี toast แม้แต่ตัวเดียว**)

**ข้อความล้มเหลว:** บล็อกกลางจอแทนที่ส่วนสถิติทั้งหมด: ไอคอน "⚠️" ตัวใหญ่ + "โหลดข้อมูลไม่สำเร็จ" + "อาจเป็นเน็ตหลุดหรือเซสชันหมดอายุ — ข้อมูลยังอยู่ในระบบ" + ปุ่มแบบ ghost ขนาดเล็ก "ลองใหม่"

### 3.2 เปลี่ยนช่วงวันที่

**ขั้นตอน:**
1. กดช่อง "ตั้งแต่วันที่" → ปฏิทินของ OS/เบราว์เซอร์เด้งขึ้นมา (ไม่ใช่ modal ที่ทีมออกแบบ — หน้าตาต่างกันไปตาม iOS / Android / Windows)
2. เลือกวัน → ระบบรับค่าทันที **ไม่มีปุ่ม "ค้นหา" หรือ "ใช้ตัวกรอง"** — เป็น live filter
3. ระบบยิงคำขอใหม่อัตโนมัติ → ตัวหมุน "กำลังโหลด…" → ตัวเลขและกราฟอัปเดต
4. ทำแบบเดียวกันกับช่อง "ถึงวันที่"
5. ถ้าเลือกจนช่วงไม่ถูกต้อง → ระบบหยุดดึงข้อมูล และ**ส่วนสถิติ/กราฟ/วิธีจัดส่งหายไปทั้งหมด** เหลือแค่การ์ดวันที่ + ข้อความแดง + การ์ดค้นหาออเดอร์

**กฎการตรวจสอบ:**
- ช่วงถูกต้องเมื่อ: กรอกครบทั้ง 2 ช่อง **และ** วันเริ่ม ≤ วันสิ้นสุด
- ช่อง "ตั้งแต่วันที่" ถูกจำกัดไม่ให้เลือกเกินค่าใน "ถึงวันที่" (เบราว์เซอร์บังคับให้)
- ช่อง "ถึงวันที่" ถูกจำกัดไม่ให้เลือกก่อนค่าใน "ตั้งแต่วันที่"
- **ไม่จำกัดความยาวช่วง** — เลือกเป็นปีก็ได้ แต่กราฟจะเรนเดอร์แค่ 92 วันแรกเงียบๆ ขณะที่การ์ดสถิติยังนับครบทุกวัน (ดูข้อ 8)
- **ไม่มีปุ่มลัดช่วงวันที่** (วันนี้ / 7 วัน / เดือนนี้) — ต้องกดปฏิทินเลือกเองทุกครั้ง

**ข้อความสำเร็จ:** ไม่มี — ตัวเลขและกราฟอัปเดตแทนที่หลังตัวหมุนหายไป

**ข้อความล้มเหลว (ช่วงผิด):** ข้อความสีแดงตัวเล็กใต้ช่องวันที่: **"ช่วงวันที่ไม่ถูกต้อง — วันเริ่มต้องไม่เกินวันสิ้นสุด"** — และเนื้อหาสถิติหายไปเฉยๆ ไม่มีตัวหมุน ไม่มี empty state ไม่มี placeholder

### 3.3 อ่านกราฟยอดขายรายวัน / หาวันขายดีสุด

**ขั้นตอน:**
1. ดูป้ายมุมขวาบนของการ์ดกราฟ — ป้ายพื้นเหลืองอ่อนตัวหนังสือน้ำตาล: "🏆 วันขายดีสุด: จ. 14 ก.ค. · ฿12,345" (แสดงเฉพาะเมื่อมีอย่างน้อย 1 วันที่ขายได้มากกว่า 0)
2. มองหาแท่ง**สีเหลืองอำพัน** = วันขายดีสุด. แท่งวันอื่นเป็น**สีแบรนด์เขียวเข้ม**. เลขวันที่ใต้แท่งวันขายดีสุดเป็นตัวหนาสีเหลืองเข้ม ส่วนวันอื่นเป็นสีเทาอ่อนปกติ
3. **บนเดสก์ท็อป:** เอาเมาส์ชี้ที่คอลัมน์แท่ง → tooltip ของเบราว์เซอร์แสดง "วันจันทร์ 14 ก.ค. · ฿1,234"
4. **บนมือถือ: แตะแท่งแล้วไม่ขึ้นอะไรเลย** — อ่านยอดรายวันไม่ได้ เห็นแค่ความสูงแท่ง (จุดอ่อนสำคัญ ดูข้อ 8)
5. ถ้าช่วงยาวเกิน 16 วัน → กราฟกว้างเกินจอ ต้องปัดซ้าย-ขวา **โดยไม่มีสัญญาณบอกว่าเลื่อนได้**

**กฎการตรวจสอบ:**
- วันที่ยอด = 0 → แสดงเป็น**ขีดเทาบางๆ 2px ที่ก้นกราฟ** ไม่ใช่ช่องว่าง (เพื่อคงความต่อเนื่องของวัน)
- แท่งที่มียอด **สูงอย่างน้อย 4% ของแท่งสูงสุดเสมอ** — ยอดน้อยๆ ต้องยังมองเห็น
- ชื่อวันย่อภาษาไทยใต้เลขวันที่ แสดงเฉพาะเมื่อช่วง ≤ 10 วัน
- กราฟเรนเดอร์ได้สูงสุด 92 แท่ง
- การจัดยอดเข้าวันใช้เขตเวลา Asia/Bangkok เสมอ

**ข้อความสำเร็จ:** ไม่มีข้อความ — กราฟปรากฏเฉยๆ

**ข้อความล้มเหลว (ไม่มียอดเลยในช่วง):** ป้ายวันขายดีสุดหายไป และกราฟทั้งหมดถูกแทนด้วย empty state จัดกลาง: ไอคอน "📊" ตัวใหญ่ + **"ไม่มียอดขายในช่วงนี้"** — **ไม่มีคำอธิบายเสริม (hint) ใดๆ**

### 3.4 ค้นหาออเดอร์ด้วยเลขออเดอร์ + ดู timeline สถานะ

**ขั้นตอน:**
1. เลื่อนลงล่างสุดของหน้า → การ์ด "ค้นหาออเดอร์"
2. พิมพ์เลขออเดอร์ในช่องที่มีข้อความตัวอย่างจางๆ ว่า "เลขออเดอร์ เช่น ORD-260714-001"
3. ปุ่ม "ค้นหา" เปลี่ยนจากสถานะกดไม่ได้ (จาง) เป็นกดได้ทันทีที่มีตัวอักษรที่ไม่ใช่ช่องว่าง
4. กดปุ่ม "ค้นหา" **หรือกด Enter ในช่องกรอก** (ทั้งสองทางทำงานเหมือนกัน)
5. ระบบแสดงตัวหมุนพร้อมข้อความ "กำลังค้นหา…" ใต้ฟอร์ม
6. ระบบค้นออเดอร์ก่อน แล้วจึงดึงประวัติสถานะของออเดอร์นั้นตามมา
7. **เจอ** → กล่องผลลัพธ์แสดง: เลขออเดอร์ + ป้ายสถานะ + ป้ายวิธีจัดส่ง + ชื่อ/เบอร์ผู้รับ + ที่อยู่ + ยอดรวมพร้อมรายละเอียด + timeline สถานะ
8. **ไม่เจอ** → empty state ไอคอน "🔍"
9. ค้นใหม่: แก้ข้อความแล้วกด "ค้นหา" อีกครั้ง — **การพิมพ์เฉยๆ ไม่ล้างผลเดิม** ผลเก่าค้างอยู่จนกว่าจะกดค้นหา

**กฎการตรวจสอบ:**
- ปุ่ม "ค้นหา" กดไม่ได้เมื่อช่องว่างหรือมีแต่ช่องว่าง
- ระบบตัดช่องว่างหน้า-หลังออกก่อนค้น
- **ไม่บังคับรูปแบบ ORD-xxxxxx-xxx** ไม่จำกัดความยาว ไม่มี pattern — พิมพ์อะไรก็ค้นได้
- ค้นแบบ**ตรงบางส่วน ไม่แคร์ตัวพิมพ์เล็ก/ใหญ่** — พิมพ์ "001" ก็เจอ
- **คืนผลได้แค่ 1 รายการเท่านั้น** — ถ้าตรงหลายออเดอร์ จะแสดงเฉพาะออเดอร์ที่สร้างล่าสุด **และไม่มีข้อความบอกว่ามีผลอื่นอีก** (ดูข้อ 8)
- **ผลค้นหาไม่ผูกกับช่วงวันที่ด้านบนเลย** — ค้นทั่วทั้งฐานข้อมูล

**ข้อความสำเร็จ:** กล่องผลลัพธ์ปรากฏใต้ฟอร์ม — ไม่มี toast ไม่มี modal ไม่มีการย้ายหน้า

**ข้อความล้มเหลว:**
- **ไม่เจอ:** ไอคอน "🔍" + **"ไม่พบออเดอร์"** + คำอธิบายตัวเล็กที่ใส่คำค้นเข้าไป ตามรูปแบบ **`ไม่มีเลขออเดอร์ที่ตรงกับ "{คำค้น}"`** เช่น `ไม่มีเลขออเดอร์ที่ตรงกับ "ORD-999"`
- **ระบบพัง:** ไอคอน "⚠️" + "โหลดข้อมูลไม่สำเร็จ" + "อาจเป็นเน็ตหลุดหรือเซสชันหมดอายุ — ข้อมูลยังอยู่ในระบบ" + ปุ่ม "ลองใหม่"
- **เจอออเดอร์แต่ไม่มีประวัติสถานะ:** ข้อความบรรทัดเดียวตัวเล็กสีเทาอ่อนในกล่องผลลัพธ์: **"ยังไม่มีประวัติสถานะ"** (ไม่ใช่ empty state แบบมีไอคอน)

### 3.5 กู้คืนเมื่อโหลดข้อมูลไม่สำเร็จ

**ขั้นตอน:**
1. เมื่อดึงข้อมูลล้มเหลว → บล็อก error แทนที่เฉพาะส่วนที่พัง
2. กดปุ่ม "ลองใหม่" → ระบบดึงข้อมูลส่วนนั้นใหม่
3. ระหว่างดึงใหม่ → กลับไปแสดงตัวหมุน → สำเร็จก็แสดงข้อมูล / ล้มเหลวก็แสดงบล็อก error เดิมซ้ำ

**กฎการตรวจสอบ:**
- **บล็อก error ของส่วนสถิติกับของส่วนค้นหาออเดอร์แยกกันคนละตัว** — พังส่วนเดียว อีกส่วนยังใช้งานได้ปกติ ต้องออกแบบให้ทั้งสองอยู่ร่วมกันบนจอเดียวกันได้
- ปุ่ม "ลองใหม่" มีเสมอในหน้านี้ (ทั้ง 2 จุดเปิดใช้)

**ข้อความสำเร็จ:** ข้อมูลกลับมาแสดงแทนบล็อก error

**ข้อความล้มเหลว:** บล็อก error เดิมแสดงซ้ำ — **ไม่มี toast และไม่แสดงข้อความ error ดิบจากระบบหลังบ้านให้ผู้ใช้เห็นเลย** (ข้อความจริงถูกกลืนไป)

---

### 3.6 พยายามเข้าหน้านี้โดยไม่ใช่แอดมิน (permission-denied)

**ขั้นตอน:**
1. ผู้ใช้ role อื่นเปิด URL `/admin/analytics` ตรงๆ
2. ระบบเด้งไปหน้าแรกของ role นั้นทันที **โดยไม่แสดงหน้าจอนี้แม้เสี้ยววินาที**
3. **ไม่มีข้อความ "ไม่มีสิทธิ์" ไม่มี toast ไม่มีหน้า 403** — เงียบสนิท
4. ในการใช้งานปกติ role อื่นจะไม่เจอกรณีนี้เลย เพราะเมนู "สถิติ" ไม่ถูกแสดงให้เห็นตั้งแต่แรก

**กฎการตรวจสอบ:** ต้องมี role `admin` เท่านั้น

**ข้อความสำเร็จ/ล้มเหลว:** ไม่มีข้อความใดๆ ทั้งสิ้น

## 4. สถานะของหน้า

| สถานะ | ผู้ใช้เห็นอะไร | ข้อความจริง |
|---|---|---|
| **ปกติ** | การ์ดวันที่ + การ์ดสถิติ 3 ใบ + กราฟแท่ง + การ์ดวิธีจัดส่ง + การ์ดค้นหา | — |
| **กำลังโหลด (ส่วนสถิติ)** | การ์ดวันที่และการ์ดค้นหายังอยู่; ส่วนสถิติ+กราฟ+วิธีจัดส่ง **ถูกแทนทั้งหมด**ด้วยตัวหมุนเล็ก (จุดหมุน 4×4 ขอบเทา หัวสีแบรนด์) จัดกลาง มีระยะบน-ล่างพอสมควร. **ไม่มี skeleton ที่ไหนเลยในระบบ** | "กำลังโหลด…" |
| **กำลังโหลด (ส่วนค้นหา)** | ตัวหมุนแบบเดียวกันใต้ฟอร์มค้นหา แทนที่ผลลัพธ์เดิม | "กำลังค้นหา…" |
| **ว่าง — ไม่มียอดขายในช่วง** | กราฟหายไป แทนด้วย empty state จัดกลาง (ไอคอน 📊 ตัวใหญ่ + หัวข้อ ไม่มี hint); ป้ายวันขายดีสุดหายไป; การ์ดสถิติยังแสดงแต่เป็น "฿0" / "0" / "—"; การ์ดวิธีจัดส่ง**หายไปทั้งใบ** | "ไม่มียอดขายในช่วงนี้" |
| **ว่าง — ค้นหาไม่พบ** | empty state ใต้ฟอร์ม ไอคอน 🔍 + หัวข้อ + คำอธิบายตัวเล็กที่ใส่คำค้นเข้าไป | "ไม่พบออเดอร์" / `ไม่มีเลขออเดอร์ที่ตรงกับ "{คำค้น}"` |
| **ว่าง — ยังไม่ได้ค้นหา** | ใต้ฟอร์มค้นหา **ไม่แสดงอะไรเลย** — ไม่มี empty state ชวนให้ค้นหา ไม่มีข้อความแนะนำ | (ไม่มีข้อความ) |
| **ว่าง — ออเดอร์ที่เจอไม่มีประวัติสถานะ** | ข้อความบรรทัดเดียวตัวเล็กสีเทาอ่อน แทนที่ timeline (ไม่มีไอคอน ไม่มีการจัดกลาง) | "ยังไม่มีประวัติสถานะ" |
| **ว่าง — ช่วงวันที่ไม่ถูกต้อง** | ส่วนสถิติทั้งหมด**หายไปเฉยๆ** — จอโล่งผิดปกติ เหลือแค่การ์ดวันที่ (พร้อมข้อความแดง) กับการ์ดค้นหา | "ช่วงวันที่ไม่ถูกต้อง — วันเริ่มต้องไม่เกินวันสิ้นสุด" |
| **ผิดพลาด (โหลด/ค้นหาพัง)** | บล็อกจัดกลาง: ไอคอน ⚠️ ตัวใหญ่ + หัวข้อตัวหนา + คำอธิบายตัวเล็กสีเทาอ่อน + ปุ่ม ghost ขนาดเล็ก. แทนที่**เฉพาะส่วนที่พัง** | "โหลดข้อมูลไม่สำเร็จ" / "อาจเป็นเน็ตหลุดหรือเซสชันหมดอายุ — ข้อมูลยังอยู่ในระบบ" / ปุ่ม: "ลองใหม่" |
| **เตือน (ออเดอร์ ≥ 1,000)** | แถบเหลืองอ่อนบางๆ เหนือการ์ดสถิติ — **ไม่ใช่ error ข้อมูลยังแสดงตามปกติ** | "⚠️ ช่วงนี้มีออเดอร์จำนวนมาก ตัวเลขอาจแสดงไม่ครบ — ลองย่อช่วงวันที่ให้แคบลง" |
| **กำลังส่ง (busy/disabled)** | **หน้านี้อ่านอย่างเดียว 100% ไม่มีการบันทึกข้อมูลใดๆ** → ไม่มีปุ่มที่เข้าสถานะ "กำลังบันทึก…" เลย. สถานะ disabled เดียวที่มีคือปุ่ม "ค้นหา" ตอนช่องว่าง — ปุ่มยังเป็นสีแบรนด์แต่จางลงครึ่งหนึ่ง | (ไม่มีข้อความ) |
| **permission-denied** | ไม่เห็นหน้านี้เลย — เด้งออกทันที ไม่มีข้อความ ไม่มีหน้า 403 และเมนู "สถิติ" ไม่ถูกแสดงตั้งแต่แรก | (ไม่มีข้อความ) |

## 5. ข้อความไทยทั้งหมดบนหน้านี้

> **ทุกข้อความในตารางนี้คือ copy จริงในระบบ ห้ามคิดข้อความไทยขึ้นเองแม้แต่คำเดียว** ข้อความที่ประกอบขึ้นจากข้อมูลจริง แสดงเป็นรูปแบบ + ตัวอย่าง

| ที่ใช้ | ข้อความจริง (verbatim) |
|---|---|
| หัวข้อหน้า + ชื่อเมนู (ไอคอน 📈) | "สถิติ" |
| ป้ายกำกับช่องวันที่ซ้าย | "ตั้งแต่วันที่" |
| ป้ายกำกับช่องวันที่ขวา | "ถึงวันที่" |
| ข้อความแดงเมื่อช่วงวันที่ผิด | "ช่วงวันที่ไม่ถูกต้อง — วันเริ่มต้องไม่เกินวันสิ้นสุด" |
| แถบเตือนออเดอร์เยอะ | "⚠️ ช่วงนี้มีออเดอร์จำนวนมาก ตัวเลขอาจแสดงไม่ครบ — ลองย่อช่วงวันที่ให้แคบลง" |
| การ์ดสถิติใบ 1 — ป้ายกำกับ | "ยอดขายรวม" |
| การ์ดสถิติใบ 1 — คำอธิบายใต้ตัวเลข | "เฉพาะออเดอร์ที่ชำระแล้ว" |
| การ์ดสถิติใบ 2 — ป้ายกำกับ | "จำนวนออเดอร์" |
| การ์ดสถิติใบ 2 — คำอธิบายใต้ตัวเลข | "= จำนวนบ้าน" |
| การ์ดสถิติใบ 3 — ป้ายกำกับ | "เฉลี่ย/ออเดอร์" |
| การ์ดสถิติใบ 3 — ค่าเมื่อไม่มีออเดอร์ | "—" (em-dash) |
| หัวข้อการ์ดกราฟ | "ยอดขายรายวัน" |
| ป้ายวันขายดีสุด (รูปแบบ) | `🏆 วันขายดีสุด: {วันย่อ วันที่ เดือนย่อ} · {฿ยอด}` — ตัวอย่าง: "🏆 วันขายดีสุด: จ. 14 ก.ค. · ฿12,345" |
| Tooltip บนแท่งกราฟ (รูปแบบ, เดสก์ท็อปเท่านั้น) | `{วันเต็ม วันที่ เดือนย่อ} · {฿ยอด}` — ตัวอย่าง: "วันจันทร์ 14 ก.ค. · ฿1,234" |
| Empty state กราฟ — หัวข้อ (ไอคอน 📊) | "ไม่มียอดขายในช่วงนี้" |
| หัวข้อการ์ดวิธีจัดส่ง | "แยกตามวิธีจัดส่ง" |
| ป้ายวิธีจัดส่ง (รูปแบบ) | `{ชื่อไทยวิธีส่ง} · {n} ออเดอร์ · {฿ยอดรวม}` — ตัวอย่าง: "ส่งปกติ (ไรเดอร์) · 12 ออเดอร์ · ฿12,345" |
| ชื่อวิธีจัดส่ง #1 | "ส่งปกติ (ไรเดอร์)" |
| ชื่อวิธีจัดส่ง #2 | "ด่วน (Grab)" |
| ชื่อวิธีจัดส่ง #3 | "นอกเขต (Lalamove)" |
| ชื่อวิธีจัดส่ง #4 | "พัสดุ (Kerry)" |
| หัวข้อการ์ดค้นหา | "ค้นหาออเดอร์" |
| ข้อความตัวอย่างในช่องค้นหา (placeholder) | "เลขออเดอร์ เช่น ORD-260714-001" |

## 6. ข้อมูลที่แสดง

| ฟิลด์ | รูปแบบการแสดงผล | ตัวอย่างจริง |
|---|---|---|
| **ยอดขายรวม** | ผลรวมยอดสุทธิของออเดอร์ทั้งหมดในช่วง. สัญลักษณ์ ฿ นำหน้าติดตัวเลข **ไม่มีทศนิยม** มีตัวคั่นหลักพัน (จัดรูปแบบไทย) | "฿12,345" |
| **จำนวนออเดอร์** | จำนวนเต็ม จัดรูปแบบไทย มีตัวคั่นหลักพัน **ไม่มีคำว่า "ออเดอร์" ต่อท้าย** | "1,234" |
| **เฉลี่ย/ออเดอร์** | ยอดรวม ÷ จำนวนออเดอร์ **ปัดเป็นจำนวนเต็มบาท** แล้วจัดรูปแบบเงิน. ถ้าไม่มีออเดอร์เลย → em-dash | "฿1,250" หรือ "—" |

| **เลขวันที่ใต้แท่ง** | เลขวันในเดือนเปล่าๆ ไม่มีเดือน ไม่มีปี ขนาด 10px | "14" |
| **ชื่อวันใต้เลขวันที่** | ชื่อวันย่อภาษาไทย ขนาด 9px — แสดงเฉพาะช่วง ≤ 10 วัน | "จ." "อ." "พ." |

**เรื่องเขตเวลาและปฏิทิน — สำคัญ:**
- การจัดยอดเข้าวัน (bucket) และการกรองช่วงวันที่ ใช้ **วันธุรกิจไทย (Asia/Bangkok, +07:00) เสมอ** — ตัวเลขไม่เพี้ยนแม้เปิดจากต่างประเทศ
- **แต่** ค่าเริ่มต้นของช่องวันที่ (วันที่ 1 ของเดือน / วันนี้) คำนวณจาก**เวลาเครื่องเบราว์เซอร์** ไม่ใช่เวลาไทย — เปิดจากต่างโซนเวลาอาจได้ค่าเริ่มต้นเหลื่อมไป 1 วัน
- วันที่ทั้งหมดจัดรูปแบบด้วย locale ไทย → **ปีจะออกมาเป็น พ.ศ.** และเดือนเป็นชื่อไทย. แต่จุดที่แสดงบนหน้านี้**ไม่มีจุดไหนแสดงปีเลย** จึงไม่เห็นผลนี้ (ยกเว้นในปฏิทินของเบราว์เซอร์เอง ซึ่งควบคุมไม่ได้)

## 7. ข้อมูลอัปเดตยังไง

**ไม่มี realtime และไม่มี polling — ทั้งสองอย่างเลย.** ไม่มีการเชื่อมต่อค้างไว้ ไม่มีการดึงซ้ำตามเวลา

**ข้อมูลรีเฟรชเฉพาะ 4 กรณีนี้เท่านั้น:**
1. เปิดหน้าครั้งแรก
2. เปลี่ยนค่าในช่อง "ตั้งแต่วันที่" หรือ "ถึงวันที่"
3. กดปุ่ม "ค้นหา" ด้วยคำค้นใหม่
4. กดปุ่ม "ลองใหม่" ในบล็อก error

**การตั้งค่าระดับระบบที่ส่งผลต่อดีไซน์:**
- **สลับแท็บออกไปแล้วกลับมา → ไม่ดึงข้อมูลใหม่** ตัวเลขค้างอยู่เหมือนเดิม
- ข้อมูลถือว่ายังสดภายใน **10 วินาที** — กลับเข้าหน้านี้ภายใน 10 วินาทีจะไม่เห็นตัวหมุนซ้ำ

**ผลต่อดีไซน์ — สิ่งที่ต้องแก้:**
- ตัวเลขบนหน้านี้เป็น **snapshot ณ ตอนโหลด ไม่ใช่ live dashboard** — แอดมินอาจนั่งดูตัวเลขที่เก่า 2 ชั่วโมงแล้วโดยไม่รู้ตัว
- **ปัจจุบันไม่มีป้ายบอกว่า "อัปเดตล่าสุดเมื่อ..." และไม่มีปุ่มรีเฟรชด้วยตัวเอง** → **ดีไซน์ใหม่ควรเพิ่มทั้งสองอย่าง** ถ้าจะเพิ่มปุ่มรีเฟรช พื้นที่ว่างฝั่งขวาของหัวข้อหน้ารองรับอยู่แล้ว (โครงสร้างหัวข้อหน้ามีช่องสำหรับปุ่มแต่หน้านี้ไม่ได้ใช้)
- **ระหว่างดึงข้อมูลใหม่ ตัวเลขและกราฟเดิมหายไปกลายเป็นตัวหมุนทั้งบล็อก** → เปลี่ยนวันที่บ่อยๆ เนื้อหาจะกระพริบหาย-มา. ดีไซน์ใหม่ควรพิจารณาให้ค่าเดิมค้างไว้พร้อมชั้นทับจางๆ แทน

## 8. สิ่งที่ designer ต้องรู้

### 🔴 ห้ามแตะ — ผูกกับกติกาธุรกิจหรือเจตนาของเจ้าของ

- **นิยามตัวเลข: นับเฉพาะออเดอร์ที่ "สถานะ ≠ ยกเลิก" และ "ชำระแล้ว" เท่านั้น** — กฎนี้ใช้กับ**ทุก**ตัวเลขและกราฟบนหน้า ห้ามเปลี่ยนนิยาม
- **คำอธิบาย "= จำนวนบ้าน" ใต้การ์ด "จำนวนออเดอร์" ห้ามตัดทิ้ง** — เป็นภาษาธุรกิจของร้าน (1 ออเดอร์ = 1 บ้านที่ต้องไปส่ง) แอดมินคิดเป็นหน่วย "บ้าน" ไม่ใช่ "ออเดอร์"
- **ข้อความไทยทุกคำในข้อ 5 ห้ามเขียนใหม่** — ถ้าเห็นว่าคำไหนควรแก้ ให้เสนอ อย่าเปลี่ยนเอง
- **เจตนาของเจ้าของ (จากเอกสารกฎธุรกิจ `Ready.md` §3.9): "สถิติคร่าวๆ ห้าม over-engineer"** — เขียนกำกับไว้ในโค้ดชัดเจน. **อย่าเปลี่ยนหน้านี้เป็น BI dashboard เต็มรูปแบบ** (อย่าเพิ่มกราฟเส้น, เปรียบเทียบช่วง, ตัวกรองซ้อน, export CSV, drill-down) โดยไม่ถามเจ้าของก่อน. งานหลักคือ **ทำสิ่งที่มีอยู่ให้อ่านง่ายขึ้น ไม่ใช่เพิ่มของ**
- **ป้ายวิธีจัดส่งเรียงตามลำดับตายตัว** (ไรเดอร์ → Grab → Lalamove → Kerry) และแสดงเฉพาะประเภทที่มีออเดอร์จริง — ห้ามเรียงตามยอด
- **หน้านี้อ่านอย่างเดียว 100%** ไม่มีการบันทึกอะไรได้เลย — ไม่ต้องออกแบบ confirm dialog, ปุ่มบันทึก, หรือสถานะ "กำลังบันทึก…" ใดๆ. ผู้ใช้กดอะไรผิดก็ไม่มีอะไรพัง

### 🟡 ปรับได้ — จุดอ่อนที่ควรถูกแก้ในดีไซน์ใหม่ (เรียงตามความสำคัญ)

1. **[สำคัญสุด] บนมือถืออ่านยอดรายวันไม่ได้เลย** — tooltip ใช้กลไก hover ของเบราว์เซอร์ ซึ่งบนมือถือแตะแล้วไม่ขึ้น. แอดมินที่เปิดจากมือถือเห็นแค่ความสูงแท่ง ไม่รู้ว่าวันนั้นขายได้เท่าไหร่. **ต้องมีวิธีดูค่าด้วยการแตะ** (แตะแล้วเผยตัวเลข, หรือแสดงตัวเลขบนหัวแท่งไปเลยเมื่อจำนวนแท่งน้อย)
2. **กราฟตัดที่ 92 วันแบบเงียบ** — เลือกช่วง 1 ปี → การ์ดสถิติ**นับครบทุกวัน** แต่กราฟ**แสดงแค่ 92 วันแรก โดยไม่บอกผู้ใช้เลย** → ตัวเลขกับกราฟขัดแย้งกันเงียบๆ. ต้องจัดการ (จำกัดช่วงที่เลือกได้, หรือแจ้งเตือนแบบเดียวกับแถบเหลือง, หรือรวมยอดเป็นรายสัปดาห์เมื่อช่วงยาว)
3. **ไม่มีปุ่มลัดช่วงวันที่เลย** (วันนี้ / 7 วัน / เดือนนี้ / เดือนที่แล้ว) — ทั้งที่เป็นสิ่งที่แอดมินทำซ้ำทุกวัน ต้องเปิดปฏิทินเลือกเองทุกครั้ง. **โอกาสปรับปรุงที่ชัดเจนและได้ผลมากที่สุดต่อความพยายาม**
4. **ช่วงเริ่มต้น (1 เดือน ≈ 30 แท่ง ≈ 660px) กว้างเกินจอมือถือแน่นอน** ต้องปัดแนวนอน **และไม่มีสัญญาณบอกว่าเลื่อนได้เลย** (ไม่มีเงา ไม่มีลูกศร) — ผู้ใช้อาจไม่รู้ว่ามีข้อมูลต่อทางขวา. ควรออกแบบให้มุมมองเริ่มต้นพอดีจอ
5. **ไม่มี "อัปเดตล่าสุดเมื่อ..." และไม่มีปุ่มรีเฟรช** — ดูข้อ 7
6. **เนื้อหากระพริบตอนเปลี่ยนวันที่** — ตัวเลขเดิมหายเป็นตัวหมุนทั้งบล็อก ควรให้ค่าเดิมค้างไว้พร้อมชั้นทับจางๆ
7. **ช่วงวันที่ผิด → เนื้อหาหายเฉยๆ จอโล่งผิดปกติ** ไม่มี placeholder ไม่มี empty state เหลือแค่การ์ดวันที่กับการ์ดค้นหาลอยอยู่
8. **ค้นหาคืนผลได้แค่ 1 รายการ แต่ค้นแบบตรงบางส่วน** — พิมพ์ "001" อาจตรงหลายออเดอร์ แต่โชว์แค่รายการล่าสุด **และไม่บอกว่ามีผลอื่นอีก** → แอดมินอาจเข้าใจผิดว่ามีออเดอร์เดียว **แล้วบอกข้อมูลผิดให้ลูกค้าทางโทรศัพท์**. ควรพิจารณาแสดงเป็นรายการผลลัพธ์ (แต่เป็นการเพิ่มของ — ต้องถามเจ้าของ)
9. **การ์ดค้นหาวางต่อท้ายส่วนสถิติ อาจสื่อผิดว่าอยู่ในขอบเขตวันที่เดียวกัน** — จริงๆ แล้วค้นทั่วทั้งฐานข้อมูล ไม่เกี่ยวกับช่วงวันที่ด้านบนเลย. ควรแยกให้เห็นชัดว่าเป็นคนละเรื่อง
10. **คำอธิบาย "เฉพาะออเดอร์ที่ชำระแล้ว" อยู่แค่บนการ์ด "ยอดขายรวม" ใบเดียว** ทั้งที่กฎนี้ใช้กับ**ทุก**ตัวเลขและกราฟบนหน้า → ควรย้ายไปเป็นคำอธิบายระดับหน้า (แต่ **ตัวข้อความห้ามเขียนใหม่**)
11. **ไม่มีแกน Y ไม่มี gridline ไม่มีตัวเลขบนแท่ง** — เทียบค่าข้ามแท่งได้ด้วยสายตาอย่างเดียว. **ค่าสูงสุดไม่ถูกแสดงเป็นตัวเลขที่ไหนเลย** ยกเว้นบังเอิญเป็นวันขายดีสุด (โผล่ในป้าย)
12. **จุดบน timeline เป็นสีแบรนด์ทุกจุดเท่ากันหมด ไม่แยกสีตามสถานะ** — ขัดกับป้ายสถานะด้านบนที่มีสี. ไม่สอดคล้องกัน
13. **timeline มีข้อมูล "สถานะก่อนหน้า" และ "ใครเป็นคนเปลี่ยน" อยู่แล้วแต่ไม่ได้แสดง** — ถ้าดีไซน์ใหม่อยากโชว์ "ใครเปลี่ยนสถานะ" **ข้อมูลมีพร้อมแล้ว ไม่ต้องแก้หลังบ้าน**
14. **การ์ดวันที่เป็น 2 คอลัมน์คงที่ทุกขนาดจอ** → บนจอแคบมาก (320px) ช่องเลือกวันที่ 2 ช่องจะอึดอัด
15. **ป้ายวิธีจัดส่งยาวมาก** (เช่น "นอกเขต (Lalamove) · 12 ออเดอร์ · ฿12,345") จะขึ้นบรรทัดใหม่หลายบรรทัดบนมือถือ. ไม่มีเปอร์เซ็นต์ ไม่มีสัดส่วนภาพ ไม่มีสีแยกประเภท
16. **กราฟไม่มี accessibility เลย** — แท่งเป็นกล่องเปล่า ไม่มีป้ายกำกับสำหรับโปรแกรมอ่านหน้าจอ ข้อมูลเข้าถึงได้ด้วยสายตาเท่านั้น

### ℹ️ ข้อมูลอ้างอิงทางเทคนิคที่ส่งผลต่อดีไซน์

- **สีแบรนด์:** เขียวเข้ม `#0F6E56` (แบบเข้ม `#0b5544`). พื้นหลังหน้า = เทาอ่อนมาก (slate-50). สีเน้นรอง = เหลืองอำพัน (amber) ใช้กับวันขายดีสุดและแถบเตือน
- **สีป้ายสถานะปัจจุบัน — มีปัญหา:**
  | สถานะ | สี |
  |---|---|
  | "ส่งสำเร็จ" | เขียว |
  | "ยกเลิก", "ตีกลับ" | แดง |
  | "รอชำระ" | เหลืองอำพัน |
  | **"ชำระแล้ว", "เข้ารอบ", "กำลังแพ็ค", "แพ็คแล้ว", "ออกส่ง", "กำลังส่ง", "รอค่าส่งใหม่"** | **เทาเหมือนกันหมด 7 สถานะ** |

  → **7 สถานะกลางทางแยกกันไม่ออกด้วยสีเลย** ทั้งที่เป็นสถานะที่แอดมินต้องอ่านให้ลูกค้าฟังบ่อยที่สุด. **ปรับได้ และควรปรับ**
- **โหมดสี: light เท่านั้น** ทั้งระบบไม่มี dark mode — ไม่ต้องออกแบบ
- **ฟอนต์: ใช้ฟอนต์ระบบเท่านั้น ไม่มีการโหลดเว็บฟอนต์** (system-ui / Segoe UI / Roboto / Noto Sans Thai). **ต้องระวังมาก:** กราฟใช้ตัวหนังสือขนาด 10px และ 9px กับชื่อวันภาษาไทย — เล็กเกินไปสำหรับสระและวรรณยุกต์ไทยที่ซ้อนกัน อ่านยากมากบนหน้าจอจริง
- **ไม่ได้ใช้ chart library ใดๆ** — กราฟเป็นกล่องธรรมดาปรับความสูงเป็นเปอร์เซ็นต์ ถ้าดีไซน์ใหม่ต้องการกราฟซับซ้อนขึ้น เป็นการเพิ่มของ (ต้องถามเจ้าของตามเจตนา "ห้าม over-engineer")
- **ทั้งหน้าไม่มี toast ไม่มี modal ไม่มี confirm dialog — เงียบสนิท** feedback ทุกอย่างเป็น inline. ระบบ**มี** modal ให้ใช้ (มือถือเป็นแผ่นเลื่อนขึ้นจากล่าง, จอใหญ่เป็นกล่องกลางจอ) แต่หน้านี้ไม่ได้ใช้เลย
- ปฏิทินที่เด้งขึ้นมาเป็นของ OS/เบราว์เซอร์ **ควบคุมหน้าตาไม่ได้** — ถ้าดีไซน์ใหม่ต้องการปฏิทินที่คุมได้ ต้องสร้างใหม่ทั้งตัว
