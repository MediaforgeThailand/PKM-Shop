# หน้าเช็คอินพนักงาน / บ้านของพนักงาน (`/staff`)

> พนักงานหน้างานทุกบทบาทเปิดหน้านี้ตอนเริ่มกะเพื่อเช็คอินเข้างานด้วยรูปถ่าย + พิกัด GPS และใช้แชททีมคุยกันระหว่างวัน

---

## 1. บริบทการใช้งาน

**ใครใช้:** พนักงานหน้างาน **ทุกคน ทุกบทบาท** (admin / stock / packer / rider / staff) — ไม่มีการกันสิทธิ์เข้าหน้านี้ ต่างจากหน้าอื่นในระบบที่ถูกล็อกตามบทบาท ใครที่ล็อกอินได้และมีบทบาทอย่างน้อยหนึ่งอย่างและบัญชียัง active ก็เข้าได้หมด

หน้านี้ยังเป็น "หน้าบ้าน" (หน้าแรกหลังล็อกอิน) ของพนักงานที่ไม่มีบทบาทเฉพาะทาง — คนที่มีแค่บทบาท `staff` จะเด้งมาที่หน้านี้เสมอ และจะเห็นแถบแท็บล่างที่**มีแท็บเดียว** คือ "เช็คอิน" (ไอคอน 👤)

**อุปกรณ์:** โทรศัพท์มือถือเป็นหลัก — **mobile-first แบบไม่มีข้อโต้แย้ง** เหตุผลอยู่ในกลไกของหน้าเอง:
- การถ่ายรูปเปิด**กล้องหลังของเครื่องโดยตรง** ไม่มีทางเลือกอัปโหลดจากไฟล์บนคอม (บนเดสก์ท็อปจะกลายเป็นหน้าต่างเลือกไฟล์ซึ่งผิดเจตนา)
- ต้องใช้ GPS ของเครื่อง — เดสก์ท็อปให้พิกัดหยาบมากจนไม่ผ่านรัศมีร้าน
- คนที่ใช้คือคนที่**ยืนอยู่ที่ร้านจริง ๆ**

**สถานการณ์จริง:** ยืนอยู่หน้าร้านหรือข้างรถ กลางแดด อาจถือของอยู่มืออีกข้าง ใช้มือเดียว รีบเข้ากะ เน็ต 4G อาจช้า จอสะท้อนแสง — **คอนทราสต์ต้องสูง ปุ่มต้องใหญ่ ทุกอย่างต้องอยู่ในระยะนิ้วโป้ง**

**ความถี่:** เช็คอินวันละ 1 ครั้ง (แต่ระบบไม่ได้ห้ามกดซ้ำ — ดู §8) ส่วนแชททีมเปิดดูได้ตลอดวัน

**เดสก์ท็อป:** รองรับแต่ไม่ใช่เป้าหมาย — ปัจจุบันหน้าจอกว้างจะเห็นการ์ดสองใบเรียงลงมาเต็มความกว้างจนดูโล่งมาก (ดู §8 — เป็นจุดที่ปรับได้)

---

## 2. โครงหน้า

หน้านี้**ไม่มีแท็บ ไม่มี modal ไม่มี wizard** — เป็นคอลัมน์เดียว เรียงจากบนลงล่าง 3 บล็อก มีระยะห่างสม่ำเสมอระหว่างบล็อก

### 2.0 กรอบรอบนอก (มีอยู่ทุกหน้า ไม่ใช่ของหน้านี้ แต่ต้องเผื่อที่ให้)
- **แถบหัวติดบนสุด (sticky)** — โลโก้ "PKM Shop" สีเขียวแบรนด์ + คำว่า "หลังบ้าน" (ซ่อนบนมือถือ), ชื่อผู้ใช้ (ตัดคำถ้ายาว), ปุ่ม "ออก" สูงประมาณ 52px + เผื่อ notch
- **แถบเตือนสีเหลือง (อาจโผล่)** — ถ้ายังไม่ผูก LINE จะเห็นข้อความ `"⚠️ ยังไม่ได้ผูก LINE — เพิ่ม PKM Shop OA เป็นเพื่อน แล้วพิมพ์รหัส {link_code} เพื่อรับแจ้งเตือนงาน"` (บนมือถืออาจกิน 2 บรรทัด)
- **แถบแท็บล่างแบบลอยติดขอบจอ (มือถือเท่านั้น)** — สูงประมาณ 56px + เผื่อ home indicator; เนื้อหาหลักเว้นที่ด้านล่างไว้ให้แล้ว
- เนื้อหาหลักจำกัดความกว้างสูงสุดระดับ "จอใหญ่มาก" จัดกึ่งกลาง มีระยะขอบรอบ

### 2.1 หัวข้อทักทาย
บรรทัดเดียว ตัวหนาขนาดกลาง: `"สวัสดี {ชื่อพนักงาน} 👋"` — ตัวอย่าง `"สวัสดี สมชาย 👋"` ถ้าไม่มีชื่อจะเป็น `"สวัสดี พนักงาน 👋"`
ไม่มีปุ่มใด ๆ ด้านขวา

### 2.2 การ์ด "เช็คอินเข้างาน"
การ์ดพื้นขาว ขอบมน เส้นขอบเทาอ่อน เงาบาง ๆ มีระยะห่างระหว่างบล็อกภายใน
หัวการ์ด: `"เช็คอินเข้างาน"` (ตัวหนา)

ลำดับภายในการ์ด จากบนลงล่าง:
1. **แถบผลเช็คอินรอบนี้** — โผล่เฉพาะหลังกดเช็คอินสำเร็จอย่างน้อย 1 ครั้งในเซสชันนี้เท่านั้น (รีเฟรชหน้าแล้วหาย) กล่องขอบมน พื้นสีอ่อน ตัวอักษรเน้น
2. **บล็อกรูปถ่าย** — ป้ายกำกับ `"รูปถ่ายหน้างาน (บังคับ)"` + ตัวเลือกรูป
3. **บล็อกสถานะ GPS** — ป้ายกำกับ `"ตำแหน่ง GPS (บังคับ)"` + กล่องพื้นเทาอ่อนขอบมนที่สลับเนื้อหาตาม 5 สถานะ
4. **ดรอปดาวน์กะทำงาน** — ป้ายกำกับ `"กะทำงาน"` + ฮินต์ตัวเล็กสีจางใต้ช่อง
5. **ปุ่มเช็คอินเต็มความกว้าง** + บรรทัดฮินต์ตัวเล็กสีจางใต้ปุ่มบอกว่ายังขาดอะไร
6. **เส้นคั่นบาง ๆ** แล้วตามด้วยหัวข้อย่อย `"เช็คอินล่าสุดของฉัน"` + รายการ 5 แถวล่าสุด

### 2.3 การ์ด "แชททีม"
การ์ดหน้าตาเดียวกัน หัวการ์ดเปลี่ยนตามสถานะ:
- ตอนกำลังโหลด / ผิดพลาด / ยังไม่มีห้อง → `"แชททีม"`
- เมื่อมีห้องแล้ว → `"แชททีม · ทีม"` (คั่นด้วยจุดกลาง · ตามด้วยชื่อห้อง ซึ่งปัจจุบันเป็น "ทีม" เสมอ)

ภายใน:
1. **กล่องรายการข้อความ** — สูงสุด 320px เลื่อนภายในตัวเอง ฟองข้อความของตัวเองชิดขวาพื้นสีเขียวแบรนด์ตัวอักษรขาว, ของคนอื่นชิดซ้ายพื้นเทาอ่อนตัวอักษรเข้ม + มีชื่อผู้ส่งบรรทัดบน; ทุกฟองมีเวลาตัวจิ๋วมุมขวาล่าง
2. **ฟอร์มพิมพ์** — ช่องข้อความยืดเต็มความกว้าง + ปุ่ม `"ส่ง"` ต่อท้าย อยู่ในสายเนื้อหาปกติ (**ไม่ได้ติดขอบจอ**)

## 3. ทุก flow ทีละสเต็ป

### 3.1 เช็คอินเข้างาน (flow หลักของหน้า)

**ขั้นตอน:**
1. เปิดหน้า → ระบบ**ขอตำแหน่ง GPS อัตโนมัติทันที** ไม่ต้องกดอะไร → เบราว์เซอร์เด้ง popup ขอสิทธิ์ตำแหน่ง → กล่อง GPS แสดงสปินเนอร์เล็ก + `"กำลังหาตำแหน่ง…"`
2. พร้อมกันนั้นดรอปดาวน์กะแสดง `"กำลังโหลดกะ…"` และรายการประวัติแสดง `"กำลังโหลดประวัติ…"` — ทั้งสามโหลดแยกกัน ไม่ขึ้นต่อกัน
3. ผู้ใช้กด "อนุญาต" → กล่อง GPS เปลี่ยนเป็นข้อความสีเขียว `"📍 ได้พิกัดแล้ว ±{X} ม."` (เช่น `"📍 ได้พิกัดแล้ว ±12 ม."`) พร้อมปุ่ม `"ขอใหม่"` ชิดขวา
4. กดกล่องถ่ายรูป (ข้อความ `"ถ่ายรูปเช็คอิน (บังคับ)"` พร้อมไอคอน 📷) → **เปิดกล้องหลังของเครื่องทันที** ไม่มีเมนูให้เลือกแกลเลอรี
5. ถ่ายรูป → กล่องเปลี่ยนเป็นพรีวิวรูป (สูงสุดประมาณ 224px ครอบเต็มความกว้าง) มีแถบไล่สีดำจาง ๆ ที่ขอบล่างและปุ่ม `"ถ่ายใหม่"` ทับอยู่มุมขวาล่าง
6. **(ไม่บังคับ)** เลือกกะจากดรอปดาวน์ — ค่าเริ่มต้นคือ `"— ไม่ระบุกะ —"`
7. เมื่อครบทั้งรูปและ GPS → ปุ่ม `"เช็คอิน"` ปลดล็อก และบรรทัดฮินต์สีเทาใต้ปุ่มหายไป
8. กดปุ่ม → ปุ่มเปลี่ยนเป็น `"กำลังเช็คอิน…"` และกดไม่ได้ (สีจางลงครึ่งหนึ่ง)
9. เบื้องหลัง: อัปโหลดรูปขึ้นที่เก็บไฟล์ส่วนตัว **แล้ว**ยิงคำสั่งเช็คอินไปเซิร์ฟเวอร์ — บนเน็ต 4G ขั้นนี้กินได้ 5–15 วินาที **ผู้ใช้เห็นแค่ข้อความ "กำลังเช็คอิน…" ไม่มี progress bar ไม่มี %**
10. เซิร์ฟเวอร์เทียบพิกัดกับพิกัดร้าน + รัศมีที่ร้านตั้งไว้ (ค่าเริ่มต้น 150 เมตร) → ตัดสินเป็น ในพื้นที่ / นอกพื้นที่ / ตรวจไม่ได้ (ร้านยังไม่ตั้งพิกัด) → **บันทึกลงระบบทุกกรณี**
11. แถบผลด้านบนการ์ดโผล่ขึ้น (เขียวหรือเหลือง) + toast เด้งขึ้นลอยเหนือแถบแท็บล่าง แสดง 3.6 วินาทีแล้วหายเอง
12. **รูปถูกล้างทิ้งอัตโนมัติ** → พรีวิวหาย กลับเป็นกล่องเปล่า → ปุ่มเช็คอินกลับไปกดไม่ได้ พร้อมฮินต์ `"ต้องถ่ายรูปก่อนเช็คอิน"` โผล่ใต้ปุ่ม (**ค่า GPS และกะที่เลือกยังอยู่ ไม่ถูกล้าง**)
13. รายการ `"เช็คอินล่าสุดของฉัน"` รีเฟรช แถวใหม่โผล่บนสุดพร้อม badge สถานะ
14. ปุ่มกลับเป็นข้อความ `"เช็คอิน"`

**กฎการตรวจสอบ (validation):**
| ฟิลด์ | บังคับ? | กฎ |
|---|---|---|
| รูปถ่ายหน้างาน | **บังคับ** | ต้องมีรูปจึงจะกดปุ่มได้ — ป้ายระบุ "(บังคับ)" ทั้งบนป้ายกำกับและในกล่อง |
| ตำแหน่ง GPS | **บังคับ** | ต้องอยู่สถานะ "ได้พิกัดแล้ว" เท่านั้น — สถานะกำลังหา/ถูกปฏิเสธ/ผิดพลาด กดไม่ได้ทั้งหมด |
| กะทำงาน | **ไม่บังคับ** | ฮินต์ระบุชัด `"เลือกได้ถ้ารู้กะของตัวเอง — ไม่บังคับ"`; ถ้าไม่เลือกจะไม่ส่งข้อมูลกะไปเลย |

เพิ่มเติม:
- **ไม่มีเกณฑ์ความแม่นยำขั้นต่ำ** — ค่า ±X เมตร แย่แค่ไหนก็ยอมรับ แค่โชว์ตัวเลขให้เห็น
- **ไม่มีการจำกัดขนาด/ฟอร์แมตรูป และไม่มีการย่อรูป** — อัปโหลดไฟล์ดิบจากกล้องตรง ๆ (3–8 MB)
- **ไม่มีการกันเช็คอินซ้ำ** ทั้งฝั่งเครื่องและเซิร์ฟเวอร์ — กด 10 ครั้งได้ 10 แถว
- กันกดซ้ำระหว่างกำลังส่งเท่านั้น (ปุ่ม disabled ตอน busy)
- ถ้าเลือกกะที่ไม่ใช่ของร้านนี้ เซิร์ฟเวอร์ตอบข้อความ `"ไม่พบกะที่เลือก"` (เคสนี้เกิดยากจาก UI ปกติ)
- **นอกพื้นที่ = ไม่บล็อก** — บันทึกสำเร็จเสมอ

**ข้อความสำเร็จ (verbatim):**

แถบผลในการ์ด (เลือกหนึ่งใน 3 ตามผล):
- ในพื้นที่ → `"✅ เช็คอินสำเร็จ — อยู่ในพื้นที่ร้าน"` (พื้นเขียวอ่อน ตัวอักษรเขียวเข้ม)
- นอกพื้นที่ → `"⚠️ อยู่นอกพื้นที่ร้าน — บันทึกแล้ว ไม่บล็อกการทำงาน"` (พื้นเหลืองอำพันอ่อน ตัวอักษรอำพันเข้ม)
- ร้านยังไม่ตั้งพิกัด → `"✅ บันทึกเช็คอินแล้ว (ร้านยังไม่ตั้งพิกัดสำหรับตรวจรัศมี)"` (พื้นเขียวอ่อน)

toast:
- ในพื้นที่ หรือ ร้านยังไม่ตั้งพิกัด → toast **สีเขียว** `"เช็คอินสำเร็จ"`
- นอกพื้นที่ → toast **สีเทาเข้ม (โทน info ไม่ใช่ error)** `"เช็คอินแล้ว (นอกพื้นที่ — บันทึกไว้แล้ว)"`

**ข้อความล้มเหลว (verbatim):**
- toast **สีแดง** `"เช็คอินไม่สำเร็จ: {ข้อความ error จริง}"` — เช่น `"เช็คอินไม่สำเร็จ: ไม่พบกะที่เลือก"`
- กรณีข้อผิดพลาดที่อ่านข้อความไม่ได้ → toast สีแดง `"เช็คอินไม่สำเร็จ"` เฉย ๆ
- **รูปไม่ถูกล้างเมื่อผิดพลาด** → ผู้ใช้กดเช็คอินซ้ำได้ทันทีด้วยรูปเดิม (จงใจ)

### 3.2 ขอตำแหน่ง GPS ใหม่ / แก้เคสถูกปฏิเสธสิทธิ์

**ขั้นตอน:**
1. **กรณีได้พิกัดแล้ว** — กดปุ่ม `"ขอใหม่"` ทางขวาของแถบพิกัด → กลับสู่สถานะกำลังหา → ได้พิกัดใหม่ ตัวเลข ±X ม. อัปเดต
2. **กรณีถูกปฏิเสธสิทธิ์** — เห็นข้อความแดง `"ปฏิเสธสิทธิ์ตำแหน่ง — เช็คอินไม่ได้ถ้าไม่มี GPS"` + ฮินต์ตัวเล็กสีเทา `"เปิดสิทธิ์ Location ให้เบราว์เซอร์ใน ตั้งค่า → แอป/เว็บไซต์ → ตำแหน่ง แล้วกดลองใหม่"` + ปุ่ม `"ลองใหม่"` → ผู้ใช้ต้องออกจากแอปไปเปิดสิทธิ์ในเครื่อง → กลับมากด `"ลองใหม่"`
3. **กรณีหาตำแหน่งไม่สำเร็จ** (หมดเวลา 15 วินาที / สัญญาณอ่อน / เบราว์เซอร์ไม่รองรับ) — เห็น `"หาตำแหน่งไม่สำเร็จ (สัญญาณ GPS อ่อนหรือหมดเวลา)"` + ปุ่ม `"ลองใหม่"` (**ไม่มีฮินต์สอนวิธีเปิดสิทธิ์ในเคสนี้**)
4. **กรณีเริ่มต้น (idle)** — เห็นปุ่ม `"📡 ขอตำแหน่ง"` → กด → กำลังหา (เคสนี้เห็นแทบไม่ได้ในชีวิตจริง เพราะระบบขอเองตอนเปิดหน้า จะเห็นแค่เสี้ยววินาทีแรก)

**กฎการตรวจสอบ:**
- ระบบพยายามเอง 1 ครั้งตอนเปิดหน้า ขอความแม่นยำสูง หมดเวลาที่ 15 วินาที ยอมรับพิกัดที่เครื่องแคชไว้ไม่เกิน 30 วินาที
- ถูกปฏิเสธสิทธิ์ → สถานะ "denied" (มีฮินต์สอนวิธีแก้)
- ผิดพลาดอื่น ๆ → สถานะ "error" (ไม่มีฮินต์ มีแค่ปุ่มลองใหม่)
- ไม่มีเกณฑ์ความแม่นยำขั้นต่ำ

**ข้อความสำเร็จ:** แถบเปลี่ยนเป็นข้อความสีเขียว `"📍 ได้พิกัดแล้ว ±{X} ม."` และปุ่มเช็คอินปลดล็อก (ถ้ามีรูปแล้ว) — **ไม่มี toast สำหรับ flow นี้**

**ข้อความล้มเหลว:** ข้อความแดงในกล่อง GPS ตามข้อ 2/3 ข้างบน (**ไม่ใช่ toast**) + ปุ่มเช็คอินยังกดไม่ได้ พร้อมฮินต์ใต้ปุ่ม `"ต้องได้พิกัด GPS ก่อนเช็คอิน"`

### 3.3 ส่งข้อความในแชททีม

**ขั้นตอน:**
1. พิมพ์ข้อความในช่อง placeholder `"พิมพ์ข้อความ…"` → ปุ่ม `"ส่ง"` ปลดล็อกทันทีเมื่อมีตัวอักษรที่ไม่ใช่ช่องว่าง
2. กด Enter หรือกดปุ่ม `"ส่ง"` → ปุ่ม disabled (**แต่ข้อความปุ่มยังเป็น "ส่ง" ไม่เปลี่ยน**)
3. บันทึกข้อความลงระบบ (ยิงตรงเข้าฐานข้อมูล ไม่ผ่านเซิร์ฟเวอร์กลาง)
4. สำเร็จ → **ช่องพิมพ์ถูกล้างทันที** → รายการดึงข้อมูลใหม่
5. ข้อความใหม่ปรากฏเป็นฟองสีเขียวแบรนด์ชิดขวา ไม่มีชื่อผู้ส่ง มีเวลา HH:MM มุมขวาล่าง
6. กล่องรายการ**เลื่อนลงล่างสุดอัตโนมัติ**
7. สมาชิกคนอื่นที่เปิดหน้านี้อยู่จะเห็นข้อความเด้งขึ้น**ทันที** (ไม่ต้องรอ)

**กฎการตรวจสอบ:**
- ข้อความต้องไม่ว่างหลังตัดช่องว่างหัวท้าย — ช่องว่างล้วนกดส่งไม่ได้
- **ไม่มีการจำกัดความยาวข้อความเลย** — พิมพ์ยาวเท่าไหร่ก็ได้ (ฟองกว้างสูงสุด 80% ของกล่อง ตัดคำยาวอัตโนมัติ ขึ้นบรรทัดตามที่พิมพ์)
- กันกดซ้ำระหว่างกำลังส่ง
- ช่องว่างหัว-ท้ายถูกตัดทิ้งเสมอ แต่การขึ้นบรรทัดกลางข้อความยังอยู่

**ข้อความสำเร็จ:** **ไม่มี toast** — ฟีดแบ็กมีแค่ ช่องพิมพ์ว่าง + ฟองข้อความโผล่ + เลื่อนลงล่าง
⚠️ **ไม่มี optimistic update** — ต้องรอเซิร์ฟเวอร์ตอบก่อนถึงจะเห็นฟองข้อความ บนเน็ตช้าจะรู้สึกหน่วง

**ข้อความล้มเหลว:** toast สีแดงแสดง**ข้อความ error ดิบจากฐานข้อมูลตรง ๆ เป็นภาษาอังกฤษเชิงเทคนิค** (เช่น `"new row violates row-level security policy"`) — ไม่ได้แปลเป็นไทย; มี fallback เป็น `"ส่งข้อความไม่สำเร็จ"` เฉพาะกรณีที่อ่านข้อความ error ไม่ได้
**ข้อความที่พิมพ์ไว้ยังอยู่ในช่อง** ไม่ต้องพิมพ์ใหม่

---

### 3.4 สร้างห้องแชททีม (เฉพาะบทบาท admin)

**ขั้นตอน:**
1. แอดมินเปิดหน้าในร้านที่ยังไม่มีห้องแชท → การ์ดแชทแสดง `"ยังไม่มีห้องแชททีม — สร้างห้องแรกให้ทีมคุยกันได้เลย"` + ปุ่มหลัก `"สร้างห้องแชททีม"` (**การ์ดนี้ไม่มีทั้งรายการข้อความและช่องพิมพ์**)
2. กดปุ่ม → ปุ่มเปลี่ยนเป็น `"กำลังสร้าง…"` และ disabled
3. สร้างห้องทันที — **ชื่อห้องถูกกำหนดตายตัวว่า `"ทีม"` ไม่มี modal ไม่มีช่องให้ตั้งชื่อ ไม่มีขั้นยืนยัน กดปุ่มเดียวจบ**
4. สำเร็จ → toast เขียว `"สร้างห้องแชททีมแล้ว"` → การ์ดเปลี่ยนเป็นแชทเต็มรูปแบบทันที หัวข้อ `"แชททีม · ทีม"` พร้อม empty state `"ยังไม่มีข้อความ — ทักทายทีมได้เลย"`

**กฎการตรวจสอบ:**
- ปุ่มแสดงเฉพาะบทบาท `admin` เท่านั้น — บทบาทอื่นเห็นแค่ข้อความรอ ไม่มีปุ่มใด ๆ
- กันกดซ้ำระหว่างกำลังสร้าง
- **ไม่มี confirm dialog** ทั้งที่ระบบมีให้ใช้

**ข้อความสำเร็จ:** toast สีเขียว `"สร้างห้องแชททีมแล้ว"`

**ข้อความล้มเหลว:** toast สีแดงแสดงข้อความ error ดิบจากฐานข้อมูล — fallback `"สร้างห้องแชทไม่สำเร็จ"`

---

### 3.5 ลองใหม่เมื่อโหลดข้อมูลไม่สำเร็จ (4 จุดอิสระต่อกัน)

**ขั้นตอน:**
1. **จุดที่ 1 — โหลดกะไม่สำเร็จ:** ในช่อง "กะทำงาน" กล่องแสดงผลผิดพลาดแทนดรอปดาวน์ → กด `"ลองใหม่"`
2. **จุดที่ 2 — โหลดประวัติเช็คอินไม่สำเร็จ:** ใต้หัวข้อ "เช็คอินล่าสุดของฉัน" → กด `"ลองใหม่"`
3. **จุดที่ 3 — โหลดห้องแชทไม่สำเร็จ:** การ์ดแชทยุบเหลือหัวข้อ `"แชททีม"` + กล่องผิดพลาด → กด `"ลองใหม่"`
4. **จุดที่ 4 — โหลดข้อความไม่สำเร็จ (แต่ได้ห้องแล้ว):** กล่องผิดพลาดแทนที่รายการฟอง **แต่ช่องพิมพ์ยังอยู่และยังส่งได้ปกติ** → กด `"ลองใหม่"`

**กฎการตรวจสอบ:**
- ทั้ง 4 จุดพังแยกกันอิสระ — กะพังไม่กระทบการเช็คอิน (เพราะกะไม่บังคับ), แชทพังไม่กระทบการ์ดเช็คอิน ผู้ใช้ยังเช็คอินได้ตามปกติ
- **ไม่มี auto-retry** — ต้องกดเอง
- กล่องผิดพลาดนี้จงใจแยกจากสถานะ "ว่าง" เพื่อไม่ให้ผู้ใช้เข้าใจผิดว่าไม่มีข้อมูล ทั้งที่จริงคือโหลดพัง

**ข้อความสำเร็จ:** ข้อมูลแสดงตามปกติแทนที่กล่องผิดพลาด — **ไม่มี toast**

**ข้อความล้มเหลว (verbatim ทั้งบล็อก):**
- ไอคอน `⚠️` ขนาดใหญ่
- `"โหลดข้อมูลไม่สำเร็จ"`
- `"อาจเป็นเน็ตหลุดหรือเซสชันหมดอายุ — ข้อมูลยังอยู่ในระบบ"`
- ปุ่ม `"ลองใหม่"`

---

### 3.6 กรณีเข้าหน้าไม่ได้ (permission-denied — เกิดก่อนถึงหน้านี้)

หน้านี้**ไม่มีการกันสิทธิ์ของตัวเอง** แต่ระดับแอปมีด่านที่ทำให้ผู้ใช้ไม่ถึงหน้านี้ — ดีไซเนอร์ต้องรู้ว่ามี 2 หน้าจอนี้อยู่:
1. **ยังไม่มีบทบาท** → เห็นหน้า `"รอผู้ดูแลเพิ่มสิทธิ์"` พร้อมข้อความ `"บัญชีนี้ยังไม่ได้ถูกกำหนดหน้าที่ในระบบ กรุณาให้แอดมินเพิ่มคุณเข้าทีมก่อน"` + `"รหัสของคุณ: {link_code}"` + ปุ่ม `"ออกจากระบบ"`
2. **บัญชีถูกปิด** → เห็นหน้า `"บัญชีถูกปิดใช้งาน"` พร้อมข้อความ `"บัญชีนี้ถูกปิดใช้งานโดยผู้ดูแล หากคิดว่าเป็นความผิดพลาด ติดต่อแอดมินของร้าน"`

ภายในหน้านี้ ความต่างตามบทบาทมีเพียง **จุดเดียว**: ปุ่ม `"สร้างห้องแชททีม"` เห็นเฉพาะ admin

|---|---|---|
| **ปกติ (พร้อมเช็คอิน)** | กล่องรูปมีพรีวิว, กล่อง GPS เขียว, ปุ่มเช็คอินสีเขียวแบรนด์กดได้เต็มความกว้าง, ไม่มีฮินต์ใต้ปุ่ม | `"เช็คอิน"` / `"📍 ได้พิกัดแล้ว ±12 ม."` |
| **ปกติ (ยังไม่พร้อม)** | ปุ่มเช็คอินจางกดไม่ได้ + ฮินต์ตัวเล็กสีเทาใต้ปุ่ม | `"ต้องมีรูปถ่ายและพิกัด GPS ก่อนเช็คอิน"` / `"ต้องถ่ายรูปก่อนเช็คอิน"` / `"ต้องได้พิกัด GPS ก่อนเช็คอิน"` |
| **กำลังโหลด — กะ** | สปินเนอร์กลม + ข้อความ แทนที่ดรอปดาวน์ | `"กำลังโหลดกะ…"` |
| **กำลังโหลด — ประวัติ** | สปินเนอร์กลม + ข้อความ แทนที่รายการ | `"กำลังโหลดประวัติ…"` |
| **กำลังโหลด — ห้องแชท** | การ์ดยุบเหลือหัวข้อ + สปินเนอร์ | `"แชททีม"` + `"กำลังโหลด…"` |
| **กำลังโหลด — ข้อความแชท** | สปินเนอร์แทนที่รายการฟอง แต่ช่องพิมพ์ยังอยู่ | `"กำลังโหลด…"` |
| **กำลังโหลด — GPS** | สปินเนอร์เล็ก inline ในกล่อง GPS | `"กำลังหาตำแหน่ง…"` |
| **กำลังโหลด — ทั้งแอป** | หน้าจอเปล่าจัดกึ่งกลาง ตัวอักษรสีเทา | `"กำลังโหลด…"` |
| **ว่าง — ประวัติเช็คอิน** | ข้อความบรรทัดเดียวสีจาง ชิดซ้าย ไม่มีไอคอน ไม่มีปุ่ม | `"ยังไม่มีประวัติเช็คอิน"` |
| **ว่าง — ข้อความแชท** | ข้อความบรรทัดเดียวสีจาง จัดกึ่งกลาง มีระยะห่างบน-ล่าง | `"ยังไม่มีข้อความ — ทักทายทีมได้เลย"` |
| **ว่าง — ยังไม่มีห้องแชท (admin)** | ข้อความ + ปุ่มหลัก | `"ยังไม่มีห้องแชททีม — สร้างห้องแรกให้ทีมคุยกันได้เลย"` + ปุ่ม `"สร้างห้องแชททีม"` |
| **ว่าง — ยังไม่มีห้องแชท (คนอื่น)** | ข้อความอย่างเดียว สีจางกว่า ไม่มีปุ่ม | `"ยังไม่มีห้องแชททีม — รอแอดมินสร้างห้องก่อน"` |
| **ว่าง — ไม่มีกะ active** | ⚠️ **ไม่มีสถานะรองรับ** — ดรอปดาวน์เหลือแค่ตัวเลือกเดียวโดยไม่บอกอะไรผู้ใช้ | `"— ไม่ระบุกะ —"` |
| **ผิดพลาด — โหลดข้อมูล (4 จุด)** | ไอคอน ⚠️ ใหญ่ + หัวข้อ + ฮินต์ + ปุ่ม | `"โหลดข้อมูลไม่สำเร็จ"` / `"อาจเป็นเน็ตหลุดหรือเซสชันหมดอายุ — ข้อมูลยังอยู่ในระบบ"` / `"ลองใหม่"` |
| **ผิดพลาด — GPS ถูกปฏิเสธ** | ข้อความแดงในกล่อง GPS + ฮินต์เทา + ปุ่ม (**ไม่ใช่ toast**) | `"ปฏิเสธสิทธิ์ตำแหน่ง — เช็คอินไม่ได้ถ้าไม่มี GPS"` / `"เปิดสิทธิ์ Location ให้เบราว์เซอร์ใน ตั้งค่า → แอป/เว็บไซต์ → ตำแหน่ง แล้วกดลองใหม่"` / `"ลองใหม่"` |
| **ผิดพลาด — GPS หาไม่เจอ** | ข้อความแดงในกล่อง GPS + ปุ่ม (ไม่มีฮินต์) | `"หาตำแหน่งไม่สำเร็จ (สัญญาณ GPS อ่อนหรือหมดเวลา)"` / `"ลองใหม่"` |
| **ผิดพลาด — ทำรายการ** | toast สีแดงลอยด้านล่าง 3.6 วินาที | `"เช็คอินไม่สำเร็จ: {error}"` / `"สร้างห้องแชทไม่สำเร็จ"` / ข้อความดิบภาษาอังกฤษ (แชท) |
| **กำลังส่ง — เช็คอิน** | ปุ่มจางลงครึ่ง กดไม่ได้ ข้อความเปลี่ยน **ไม่มี progress** | `"กำลังเช็คอิน…"` |
| **กำลังส่ง — สร้างห้อง** | ปุ่มจาง กดไม่ได้ ข้อความเปลี่ยน | `"กำลังสร้าง…"` |
| **กำลังส่ง — ส่งข้อความ** | ปุ่มจาง กดไม่ได้ **แต่ข้อความไม่เปลี่ยน** (ไม่สอดคล้องกับอีก 2 ปุ่ม) | `"ส่ง"` (คงเดิม) |
| **สำเร็จ — เช็คอิน** | แถบสีในการ์ด + toast + แถวใหม่ในประวัติ + กล่องรูปกลับเป็นเปล่า | ดู §3.1 |
| **permission-denied** | ไม่เกิดในหน้านี้ — ถูกกันที่ระดับแอปก่อน (ดู §3.6) | `"รอผู้ดูแลเพิ่มสิทธิ์"` / `"บัญชีถูกปิดใช้งาน"` |

## 6. ข้อมูลที่แสดง

| ฟิลด์ | รูปแบบการแสดงผล | ตัวอย่างจริง |
|---|---|---|
| ชื่อพนักงาน (หัวข้อ) | ข้อความดิบ ต่อท้ายด้วย 👋 | `สวัสดี สมชาย 👋` |
| ความแม่นยำ GPS | ตัวเลขปัดเศษเป็นจำนวนเต็ม + หน่วย "ม." นำหน้าด้วย ± | `±12 ม.` / `±47 ม.` |
| ชื่อกะ + ช่วงเวลา | `{ชื่อ} {HH:MM}–{HH:MM}` — **ตัดวินาทีทิ้ง**, คั่นด้วยขีดกลาง en dash `–` (ไม่ใช่ hyphen) | `เช้า 08:00–17:00` / `บ่าย 13:00–22:00` |
| วันที่-เวลาเช็คอิน (รายการประวัติ) | **ปฏิทินไทย ชื่อเดือนย่อภาษาไทย + เวลา 24 ชม.** โซนเวลา Asia/Bangkok เสมอ (ตายตัว) — ไม่มีปี | `16 ก.ค. 08:32` / `1 ธ.ค. 17:05` |
| สถานะ geofence (badge) | ป้ายกลมรี ตัวอักษรจิ๋วตัวหนา 3 แบบ: เขียว / เหลืองอำพัน / เทา | `✅ ในพื้นที่` / `⚠️ นอกพื้นที่` / `บันทึกแล้ว` |
| เวลาข้อความแชท | **เฉพาะ ชั่วโมง:นาที 24 ชม.** โซน Asia/Bangkok — **ไม่มีวันที่เลย** | `08:32` / `17:05` |
| ชื่อผู้ส่ง (ฟองของคนอื่น) | ข้อความดิบตัวจิ๋วตัวหนาสีเทา — fallback เป็น `ทีม` ถ้าไม่มีชื่อ | `สมหญิง` |
| เนื้อข้อความแชท | ข้อความดิบ **คงการขึ้นบรรทัดตามที่พิมพ์** + ตัดคำยาวอัตโนมัติ | — |
| ชื่อห้องแชท | ข้อความดิบ ต่อท้ายหัวการ์ดด้วยจุดกลาง | `แชททีม · ทีม` |

**หมายเหตุสำคัญเรื่องข้อมูล:**
- **หน้านี้ไม่มีตัวเลขเงินเลย** — ไม่มีการแสดงราคา/ยอดใด ๆ (ไม่ใช้ตัวจัดรูปแบบเงินบาทของระบบ)
- **ไม่มีเลขออเดอร์** — ไม่เกี่ยวข้องกับหน้านี้
- **ไม่มีการแสดงชื่อกะในรายการประวัติ** — ระบบดึงข้อมูลกะมาแล้วแต่ไม่ได้เอามาแสดง ผู้ใช้เลือกกะไปแต่ไม่มีทางเห็นว่าเช็คอินครั้งนั้นผูกกับกะไหน
- **รูปเช็คอินไม่ถูกแสดงกลับเลย** — เห็นแค่ตอนพรีวิวก่อนกดส่ง หลังส่งแล้วดูย้อนหลังไม่ได้ (รูปเก็บในที่เก็บไฟล์ส่วนตัว)
- **รายการประวัติจำกัด 5 แถวตายตัว** ใหม่สุดอยู่บน — ไม่มีปุ่มดูเพิ่ม ไม่มี pagination ไม่มีหน้าประวัติเต็ม
- **แชทดึง 50 ข้อความล่าสุด** เรียงเก่า→ใหม่ — ไม่มี infinite scroll ไม่มีปุ่มโหลดเพิ่ม เลื่อนขึ้นสุดแล้วจบ

## 7. ข้อมูลอัปเดตยังไง

**สั้น ๆ: การ์ดแชทสด — การ์ดเช็คอินนิ่ง**

| ส่วน | กลไก | ผลต่อดีไซน์ |
|---|---|---|
| **ข้อความแชท** | **Realtime** — ข้อความใหม่ของคนอื่นเด้งขึ้นทันทีโดยไม่ต้องรีเฟรช (จับเฉพาะข้อความใหม่ — การแก้ไข/ลบไม่เด้ง แต่ระบบก็ไม่มีฟีเจอร์แก้/ลบอยู่แล้ว) | ต้องออกแบบให้รองรับ**เนื้อหาที่โผล่มาเองระหว่างผู้ใช้กำลังอ่าน** — ปัจจุบันจอกระโดดลงล่างทันที (ดู §8) |
| **ข้อความแชท (สำรอง)** | **Polling ทุก 30 วินาที** เผื่อ Realtime หลุด | ไม่ต้องมีตัวบอก "อัปเดตล่าสุด" — ผู้ใช้ไม่ควรรู้สึกว่าต้องรอ |
| **รายการกะ** | **โหลดครั้งเดียว** ตอนเปิดหน้า (+ ดึงใหม่อัตโนมัติเมื่อกลับมาโฟกัสหน้าต่าง) | นิ่ง ไม่ต้องมีตัวบอกเวลา |
| **ประวัติเช็คอิน** | **โหลดครั้งเดียว** + ดึงใหม่หลังเช็คอินสำเร็จ + กด "ลองใหม่" + กลับมาโฟกัสหน้าต่าง | ไม่มี polling — ถ้าคนอื่นเช็คอินให้ (ซึ่งเป็นไปไม่ได้) จะไม่เห็น ไม่เป็นปัญหา |
| **ห้องแชท** | **โหลดครั้งเดียว** + ดึงใหม่หลังสร้างห้องสำเร็จ | ถ้าแอดมินสร้างห้องจากเครื่องอื่น พนักงานที่เปิดหน้าค้างไว้จะ**ไม่เห็นห้องจนกว่าจะรีเฟรช** |
| **GPS** | **ขอครั้งเดียวตอนเปิดหน้า** — ไม่ติดตามต่อเนื่อง (ไม่ใช่ watchPosition) | ⚠️ พิกัดที่ได้อาจเก่าถ้าผู้ใช้เปิดหน้าค้างไว้แล้วเดินไปที่อื่น — ปุ่ม "ขอใหม่" คือทางแก้เดียว **ดีไซน์ควรทำให้ปุ่มนี้เห็นชัด** หรือพิจารณาแสดงเวลาที่ได้พิกัดมา |

**สรุปเรื่องตัวบอก "อัปเดตล่าสุด":**
- **แชท: ไม่ต้องมี** — เป็น realtime อยู่แล้ว การใส่จะสร้างความสับสน
- **GPS: น่าจะควรมี** — พิกัดค้างได้จริง แต่ปัจจุบันไม่มีทั้งเวลาและตัวบอกความสด (เป็นข้อเสนอปรับปรุง ไม่ใช่ข้อกำหนดปัจจุบัน)
- **ประวัติเช็คอิน: ไม่จำเป็น** — ข้อมูลเปลี่ยนก็ต่อเมื่อผู้ใช้เองเป็นคนทำ

**ไม่มีเลยในหน้านี้:** optimistic update (ทุกอย่างรอเซิร์ฟเวอร์ตอบก่อน), skeleton screen (ใช้สปินเนอร์ล้วน), presence/typing indicator/สถานะออนไลน์

## 8. สิ่งที่ designer ต้องรู้

### 🔒 ห้ามแตะ — ผูกกับกฎธุรกิจหรือข้อจำกัดทางเทคนิค

- **GPS เป็นด่านบังคับ แต่ "นอกพื้นที่" ไม่ใช่ความล้มเหลว** — นี่คือหัวใจของกฎธุรกิจ (Ready.md §3.8): ไม่มี GPS = เช็คอินไม่ได้เลย แต่ "อยู่นอกรัศมีร้าน" = **เช็คอินได้ปกติ แค่ถูกบันทึกไว้**
  **ห้ามออกแบบให้ "นอกพื้นที่" ดูเหมือน error** — โค้ดจงใจใช้ toast โทนเทาเข้ม (info) ไม่ใช่แดง และใช้สีเหลืองอำพันไม่ใช่แดง คำว่า "ไม่บล็อกการทำงาน" ในแถบผลคือการสื่อสารเชิงนโยบายที่ตั้งใจให้พนักงานอุ่นใจ
- **รูปถ่าย + GPS บังคับทั้งคู่** — ห้ามลดเหลืออย่างใดอย่างหนึ่ง ห้ามทำให้ข้ามได้
- **การถ่ายรูปเปิดกล้องหลังโดยตรง** — ไม่มีตัวเลือกแกลเลอรี ไม่มี crop เป็นเจตนาเชิงกฎ (ต้องเป็นรูป ณ หน้างานจริง) **ห้ามเพิ่มทางเลือก "อัปโหลดจากไฟล์"** โดยไม่ถามเจ้าของ
- **ข้อความไทยทุกคำใน §5** — ห้ามแต่งใหม่เอง ถ้าต้องการข้อความใหม่ ต้องถามเจ้าของ (Ready.md เป็นแหล่งกฎธุรกิจเดียว)
- **Light mode อย่างเดียว** — ทั้งแอปประกาศ light เท่านั้น ไม่มี dark mode ไม่ต้องออกแบบ
- **ระยะสัมผัสขั้นต่ำ** — ปุ่มสูงอย่างน้อย 44px, ช่องกรอกสูงอย่างน้อย 44px และตัวอักษรขนาด 16px (**จำเป็นเพื่อกัน iOS ซูมอัตโนมัติตอนโฟกัส — ห้ามลดขนาดฟอนต์ในช่องกรอก**), แถบแท็บล่างสูงอย่างน้อย 56px, มีการเผื่อพื้นที่ notch/home indicator
- **Design token ที่ผูกอยู่** — สีแบรนด์เขียวเข้ม `#0F6E56` (สถานะ hover เข้มขึ้นเป็น `#0b5544`), พื้นหลังหน้าเทาอ่อนมาก, ตัวอักษรหลักเทาเข้มเกือบดำ; ฟอนต์ใช้ฟอนต์ระบบโดยมี **Noto Sans Thai** อยู่ในลำดับแล้ว
- **เวลาเป็นโซนกรุงเทพเสมอ (ตายตัว) และภาษาไทย** — วันที่ใช้ชื่อเดือนไทยย่อ ("16 ก.ค.") ความยาวสตริงไทยต่างจากอังกฤษ ระวังเวลาจัด layout ให้พอดีเป๊ะ
- **Toast ลอยเหนือแถบแท็บล่างบนมือถือ** (และขยับลงเมื่อจอกว้าง) อายุ 3.6 วินาที ซ้อนกันได้หลายอัน — **ดีไซน์ toast ใหม่ต้องไม่ทับแถบแท็บล่างและไม่ทับช่องพิมพ์แชท**
- **แชทมีห้องเดียวเสมอ** — ระบบดึงห้องที่เปิดใช้งานและเก่าที่สุดมาแค่ห้องเดียว ไม่มีรายการห้อง ไม่มีการสลับห้อง ชื่อห้องกำหนดตายตัวว่า "ทีม"
  **อย่าออกแบบ UI แบบ Slack / multi-channel** เว้นแต่จะขยาย scope และถามเจ้าของก่อน
- **สถานะ "ร้านยังไม่ตั้งพิกัด" ไม่ใช่ bug** — ร้านเปิดใหม่จะเจอสถานะนี้เป็นค่าเริ่มต้น ทุกการเช็คอินได้ badge เทา "บันทึกแล้ว" **ดีไซน์ต้องทำให้สถานะนี้ดูปกติ ไม่ใช่ดูเสีย**

### 🛠 ปรับได้ — จุดที่ดีไซน์ปัจจุบันอ่อนและควรถูกปรับปรุง

- **ไม่มี progress ตอนอัปโหลดรูป** ⚠️ pain point ที่ใหญ่ที่สุด — รูปจากกล้องมือถือ 3–8 MB อัปโหลดดิบ ๆ ไม่ย่อ บน 4G กินได้ 5–15 วินาที แต่ผู้ใช้เห็นแค่ปุ่มที่เขียนว่า "กำลังเช็คอิน…" ไม่มี progress bar ไม่มี % ไม่มีปุ่มยกเลิก — คนยืนกลางแดดจะไม่แน่ใจว่าค้างหรือกำลังทำงาน
- **หลังเช็คอินสำเร็จ ฮินต์ "ต้องถ่ายรูปก่อนเช็คอิน" โผล่ทันที** ⚠️ — เพราะรูปถูกล้างอัตโนมัติ ผู้ใช้เพิ่งเช็คอินสำเร็จหยก ๆ แล้วเห็นข้อความบอกว่ายังขาดรูป **อ่านเป็นสัญญาณผิดว่า "ยังทำไม่เสร็จ"** ควรทำให้แถบผลสำเร็จเด่นกว่าฮินต์นี้ หรือซ่อนฮินต์ชั่วคราวหลังสำเร็จ
- **แถบผลเช็คอินไม่คงอยู่** — รีเฟรชหน้าแล้วหายทันที ผู้ใช้ต้องดูรายการประวัติแทน; ถ้าอยากให้ "สถานะวันนี้" คงอยู่ ต้องแจ้งเป็น requirement ใหม่ (และถามเจ้าของ)
- **ไม่มีการกันเช็คอินซ้ำเลย** ทั้งฝั่งเครื่องและเซิร์ฟเวอร์ — กด 10 ครั้งได้ 10 แถว รายการประวัติอาจเต็มไปด้วยแถวเวลาใกล้กัน; ไอเดียอย่าง "วันนี้เช็คอินแล้วเมื่อ 08:32" หรือสถานะ "เข้างานอยู่" น่าสนใจ **แต่ต้องถามเจ้าของก่อน — ห้ามคิดกฎธุรกิจเอง**
- **แชทเลื่อนลงล่างสุดแบบไม่มีเงื่อนไข** ⚠️ UX bug — ถ้าผู้ใช้กำลังเลื่อนอ่านข้อความเก่าแล้วมีคนส่งข้อความใหม่เข้ามา **จอกระโดดลงล่างทันที**; ควรแก้เป็นเลื่อนเฉพาะเมื่อผู้ใช้อยู่ใกล้ล่างสุด หรือมีป้าย "ข้อความใหม่ ↓" ให้กด
- **กล่องแชทสูงแค่ 320px** — บนมือถือเห็นได้ราว 4–6 ฟองเท่านั้น ค่อนข้างอึดอัด
- **ปุ่มส่งไม่เปลี่ยนข้อความตอนกำลังส่ง** (คงเป็น "ส่ง" แค่จางลง) ต่างจากปุ่มเช็คอิน ("กำลังเช็คอิน…") และปุ่มสร้างห้อง ("กำลังสร้าง…") — ความไม่สอดคล้องที่ควรรวบให้เป็นระบบเดียว
- **empty state ทั้ง 3 จุดเป็นข้อความเปล่า ๆ** ไม่ได้ใช้คอมโพเนนต์ empty state กลางของระบบ (ที่มีไอคอน 📭 + หัวข้อ + ฮินต์) — ถ้าจะทำให้ทั้งแอปเป็นระบบเดียวกัน ควรพิจารณาใช้ตัวกลาง
- **"ไม่มีกะ active" ไม่มีสถานะรองรับเลย** — ดรอปดาวน์เหลือแค่ "— ไม่ระบุกะ —" โดยไม่บอกอะไรผู้ใช้ ดูเหมือนโหลดไม่ครบ
- **ข้อความ error ของแชท 2 จุดเป็นภาษาอังกฤษเชิงเทคนิคดิบ ๆ** (เช่น "new row violates row-level security policy") ต่างจากส่วนเช็คอินที่มีคำนำหน้าไทย — ถ้าอยาก copy ไทยล้วนทั้งหน้า ต้องแจ้ง dev
- **แถบแท็บล่างมีแท็บเดียวสำหรับพนักงานทั่วไป** — คนที่มีแค่บทบาท `staff` จะเห็นแถบเต็มความกว้างที่มีปุ่มเดียวคือ "👤 เช็คอิน" ซึ่งดูแปลกมาก ควรพิจารณาซ่อนแถบเมื่อมีแท็บเดียว หรือออกแบบเคสนี้เฉพาะ
- **บนจอเดสก์ท็อปหน้านี้โล่งมาก** ⚠️ — ไม่มี grid 2 คอลัมน์เลย การ์ดเช็คอินและการ์ดแชทเรียงลงมาเต็มความกว้างสูงสุด (กว้างมาก) เป็นโอกาสปรับปรุงที่ชัดเจน แต่ต้องไม่ทำลาย mobile-first
- **ชื่อผู้ส่งแสดงทุกฟองไม่จัดกลุ่ม** — ส่งติดกัน 5 ข้อความ = เห็นชื่อ 5 ครั้ง
- **แชทไม่มี:** avatar, เส้นคั่นวันที่, สถานะส่งแล้ว/อ่านแล้ว, typing indicator, presence, แนบรูป, ตอบกลับ, react, แก้ไข/ลบ — อย่าออกแบบสิ่งเหล่านี้เข้าไปโดยไม่ขยาย scope
- **ชื่อกะไม่ถูกแสดงในประวัติ** — ข้อมูลมีอยู่แล้วแต่ไม่ได้เอามาใช้ เป็นช่องว่างที่เติมได้ (ต้องดึงชื่อกะเพิ่ม)
- **การเข้าถึง (a11y) ยังไม่ครบ** — กล่อง GPS ไม่มีการประกาศสถานะให้ screen reader อ่านเมื่อค่าเปลี่ยน (จาก "กำลังหาตำแหน่ง…" → "ได้พิกัดแล้ว") ปุ่มส่วนใหญ่ใช้ข้อความจริงจึงพออ่านได้
- **ไม่มี i18n** — ข้อความไทยฝังในโค้ดทั้งหมด ไม่ต้องออกแบบเผื่อภาษาอื่น
