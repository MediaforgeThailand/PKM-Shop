# เงินเดือน / ค่ารอบ (`/admin/payroll`)

> แอดมินเปิดหน้านี้ตอนถึงรอบจ่ายเงิน เพื่อดูยอดค่ารอบ/ค่าคอมของพนักงานแต่ละคน แล้วยืนยันว่าโอนเงินไปแล้วพร้อมแนบรูปสลิปเป็นหลักฐาน

---

## 1. บริบทการใช้งาน

**ใครใช้:** แอดมิน (role `admin`) เท่านั้น — คนเดียวในทีมที่เห็นหน้านี้

- ฝั่งหน้าเว็บมี Guard กันไว้: ถ้าไม่ใช่ `admin` จะถูกเด้งกลับไปหน้าแรกตาม role ของตัวเอง (`stock`→หน้าแคตตาล็อก, `packer`→หน้าแพ็ก, `rider`→หน้าไรเดอร์, อื่นๆ→หน้าสตาฟ) **ไม่มีหน้าจอ "คุณไม่มีสิทธิ์" ให้เห็นเลย — เด้งออกเงียบๆ ทันที**
- เมนู "💰 เงินเดือน" ใน tab bar จะไม่โผล่ให้ role อื่นเห็นด้วยซ้ำ
- ฝั่งเซิร์ฟเวอร์ก็เช็ค role ซ้ำอีกชั้นก่อนบันทึกการโอน

**อุปกรณ์:** **Mobile-first** — เพราะ flow จริงคือแอดมินถือมือถือเครื่องเดียว สลับไป-กลับระหว่างแอปธนาคารกับแอปนี้ ถ่าย/แคปสลิปแล้วแนบกลับเข้ามา หน้านี้ถูกออกแบบเป็นคอลัมน์เดียว มี tab bar ติดล่างจอ และ modal เป็น bottom sheet บนมือถือ / กล่องกึ่งกลางจอบนจอใหญ่

**สถานการณ์จริง:** นั่งอยู่ (ไม่ใช่งานยืนกลางแดดแบบหน้าไรเดอร์) แต่ **สมาธิแตก** — ต้องเปิดแอปธนาคาร โอนเงิน แคปหน้าจอ กลับมาแอปนี้ แนบรูป ยืนยัน แล้ววนซ้ำทีละคนจนครบทีม

**ความถี่:** ต่ำมาก — สัปดาห์ละครั้ง หรือปักษ์ละครั้ง (ประเมินจากที่ระบบเก็บรอบไว้ 26 รอบล่าสุด = ประมาณครึ่งปีถ้าจ่ายรายสัปดาห์ / 1 ปีถ้าจ่ายรายปักษ์)

**ความเร่งรีบ:** ไม่รีบ แต่ **ผิดไม่ได้** — ทุกการกดปุ่มยืนยันคือการบันทึกว่า "จ่ายเงินคนนี้แล้ว" อย่างถาวร แก้ไม่ได้จากหน้านี้ และระบบจะยิงแจ้งเตือน LINE ไปหาพนักงานทันที ความเสี่ยงคือ "จ่ายซ้ำ" หรือ "ทำเครื่องหมายว่าจ่ายแล้วทั้งที่ยังไม่ได้โอน"

---

## 2. โครงหน้า

หน้านี้เป็น **คอลัมน์เดียวทั้งหมด ไม่มีแท็บภายในหน้า ไม่มี grid** การ์ดเรียงต่อกันจากบนลงล่างด้วยระยะห่างสม่ำเสมอ พื้นที่เนื้อหากว้างสุดประมาณระดับ container กลาง แต่เพราะเป็นคอลัมน์เดียว **บนจอใหญ่จึงดูโล่งมาก**

ลำดับบล็อกจากบนลงล่าง:

### 2.1 หัวข้อหน้า (แสดงเสมอ ทุกสถานะ)
บรรทัดหัวข้อตัวหนา ข้อความ **"เงินเดือน / ค่ารอบ"** — ไม่มีปุ่ม action ด้านขวา ไม่มีคำอธิบายใต้หัวข้อ แสดงค้างอยู่แม้ตอนกำลังโหลด/ผิดพลาด/ว่าง

### 2.2 ช่องเลือกรอบ (แสดงเมื่อโหลดสำเร็จและมีรอบอย่างน้อย 1 รอบ)
ฟิลด์แบบ label + dropdown ในกล่องเดียว
- Label: **"เลือกรอบ"** (ไม่มีข้อความ hint ใต้ฟิลด์)
- ตัวเลือกใน dropdown = รอบจ่ายเงิน สูงสุด 26 รอบ เรียงจากใหม่→เก่า ระบบเลือกรอบล่าสุดให้อัตโนมัติตอนเปิดหน้า
- รูปแบบข้อความในแต่ละตัวเลือก: `{วันเริ่ม} – {วันจบ} · {สถานะ}`
  ตัวอย่างจริง: `1 ก.ค. 68 – 15 ก.ค. 68 · กำลังสะสม` หรือ `16 มิ.ย. 68 – 30 มิ.ย. 68 · ปิดรอบแล้ว`
  (คำสถานะ verbatim: **"· กำลังสะสม"** / **"· ปิดรอบแล้ว"**)

### 2.3 การ์ดสรุปรอบ (แสดงคู่กับ dropdown เสมอ)
การ์ดพื้นขาว ขอบมน มี 2 บรรทัด:
- **บรรทัดบน** — ข้อความเล็กสีเทา: `รอบ {วันเริ่ม} → {วันจบ}` ตัวอย่างจริง: **"รอบ 1 ก.ค. 68 → 15 ก.ค. 68"**
- **บรรทัดล่าง** — ซ้าย/ขวาชิดขอบ:
  - ซ้าย = ป้ายสถานะ (badge) 2 แบบ: **"ปิดรอบแล้ว"** พื้นเขียวอ่อน ตัวอักษรเขียวเข้ม / **"กำลังสะสม"** พื้นเหลืองอำพันอ่อน ตัวอักษรอำพันเข้ม
  - ขวา = ยอดรวมตัวหนา รูปแบบ `รวม ฿{ตัวเลข}` ตัวอย่างจริง: **"รวม ฿12,500"**

> **กติกาของยอดรวม:** ถ้ารอบนั้นตัดแล้ว (มีรายการ payout) ยอดรวมคือผลรวมของ payout ทุกคน; ถ้ายังไม่ตัด ยอดรวมคือผลรวมยอดสะสมทุกคน — **การ์ดใบเดียวกันจึงหมายความคนละอย่างขึ้นกับสถานะรอบ** และถ้ารอบว่างจะโชว์ **"รวม ฿0"**

### 2.4 พื้นที่รายการ — มี 3 หน้าตา สลับกันตามสถานะรอบ (แสดงได้ทีละแบบเท่านั้น)

**แบบ ก — รอบตัดแล้ว: รายการ payout รายคน** (เมื่อมี payout อย่างน้อย 1 แถว)
- **ไม่มีหัวข้อกำกับกลุ่มนี้เลย** — การ์ดโผล่ต่อจากการ์ดสรุปเลย
- เรียงจากยอดมาก→น้อย ไม่มีการแบ่งหน้า ไม่มีตัวกรอง ไม่มีช่องค้นหา — แสดงทุกคนในรอบ
- แต่ละการ์ด: ซ้าย = ชื่อพนักงานตัวหนา (ยาวเกินจะตัดด้วย …) ใต้ชื่อเป็นยอดเงินตัวเล็กสีเทา; ขวา = ตัวควบคุมตามสถานะ
  - **ยังไม่โอน** → ปุ่มสีหลัก ขนาดเล็ก ข้อความ **"ยืนยันโอน + แนบสลิป"** (ไม่มี badge สถานะใดๆ — ตัวปุ่มคือสัญญาณเดียวว่าคนนี้ยังไม่ได้เงิน)
  - **โอนแล้ว** → badge พื้นเขียวอ่อน ข้อความ **"✅ โอนแล้ว"** และใต้ badge เป็นวันเวลาโอนตัวเล็กสีเทาจาง เช่น **"15/7/2568 14:30"**
- **เพิ่มเติมเฉพาะการ์ดที่โอนแล้วและมีสลิปแนบ:** ปุ่มโปร่ง (ghost) เต็มความกว้างการ์ดที่แถวล่าง ข้อความ **"ดูสลิปการโอน"** → ทำให้การ์ดที่โอนแล้ว **สูงกว่า** การ์ดที่ยังไม่โอน

**แบบ ข — รอบยังไม่ตัด: ยอดสะสมต่อคน (read-only)** (เมื่อไม่มี payout แต่มีรายการสะสม)
- มีหัวข้อกลุ่มตัวเล็กสีเทา ข้อความ verbatim: **"ยอดสะสมต่อคน (ยังไม่ตัดรอบ)"**
- การ์ดต่อคน: ชื่อซ้าย / ยอดเงินตัวหนาขวา — **ไม่มีปุ่ม ไม่มี badge กดการ์ดหรือกดชื่อไม่ทำอะไรทั้งสิ้น**

**แบบ ค — รอบว่าง:** empty state (ดู §4)

### 2.5 Modal ที่ซ้อนขึ้นมา (ไม่ใช่บล็อกในหน้า)
บนมือถือเป็น **bottom sheet** เลื่อนขึ้นจากขอบล่าง มุมบนมน เผื่อระยะ safe area; บนจอใหญ่เป็น **กล่องกึ่งกลางจอ** ขอบมนทุกมุม กว้างจำกัด สูงไม่เกิน 90% ของจอ (เลื่อนในกล่องได้) พื้นหลังดำโปร่ง — **คลิกพื้นหลัง = ปิด** และมีปุ่ม **"✕"** มุมขวาบน (มีป้ายช่วยการเข้าถึงว่า "ปิด")

1. **Modal ยืนยันโอน** — เปิดจากปุ่ม "ยืนยันโอน + แนบสลิป" (รายละเอียดใน §3.2)
2. **Modal ดูสลิป** — เปิดจากปุ่ม "ดูสลิปการโอน" (รายละเอียดใน §3.3)

> หน้านี้ **ไม่มี** dialog ถามยืนยันซ้ำ (confirm) ก่อนบันทึกการโอน แม้ระบบจะมีคอมโพเนนต์นั้นให้ใช้

## 3. ทุก flow ทีละสเต็ป

### 3.1 Flow: ดู / สลับรอบจ่ายเงิน

**ขั้นตอน**
1. แอดมินแตะเมนู **"💰 เงินเดือน"** จาก tab bar ล่าง (มือถือ) หรือแถบ nav บน (จอใหญ่)
2. หน้าเปิดขึ้น หัวข้อ "เงินเดือน / ค่ารอบ" แสดงทันที ส่วนที่เหลือเป็นวงกลมหมุน + ข้อความ **"กำลังโหลด…"**
3. โหลดเสร็จ → ระบบ **เลือกรอบล่าสุดให้อัตโนมัติ** ผู้ใช้ไม่ต้องกดอะไร
4. เห็น dropdown "เลือกรอบ" + การ์ดสรุปรอบ (ช่วงวันที่ + badge สถานะ + "รวม ฿X") + รายการด้านล่าง
5. (ถ้าต้องการดูรอบย้อนหลัง) แตะ dropdown → เลือกรอบที่ต้องการจากรายการสูงสุด 26 รอบ
6. เนื้อหาด้านล่างถูกโหลดใหม่ทันที → ระหว่างนั้นเห็น **"กำลังโหลด…"** เฉพาะพื้นที่รายการ **แต่ dropdown และการ์ดสรุปรอบยังคาอยู่ที่เดิม ไม่กระพริบหาย**

**กฎการตรวจสอบ (validation)**
- ไม่มี validation — dropdown มีแต่ตัวเลือกที่ถูกต้องเสมอ เลือกผิดไม่ได้
- ถ้าไม่มีรอบเลย → **ไม่แสดง dropdown** และไม่แสดงการ์ดสรุป (ดู empty state §4)
- จำกัดแค่ 26 รอบล่าสุด — **ไม่มีปุ่มโหลดเพิ่ม ไม่มี pagination** รอบที่เก่ากว่านั้นเข้าถึงจากหน้านี้ไม่ได้เลย

**ข้อความสำเร็จ:** ไม่มี toast — เนื้อหาเปลี่ยนตามรอบที่เลือกเงียบๆ

**ข้อความล้มเหลว (โหลดรอบไม่สำเร็จ):** แทนที่ทั้งหน้าใต้หัวข้อด้วย
> ⚠️
> **"โหลดข้อมูลไม่สำเร็จ"**
> **"อาจเป็นเน็ตหลุดหรือเซสชันหมดอายุ — ข้อมูลยังอยู่ในระบบ"**
> ปุ่มโปร่ง **"ลองใหม่"**

(dropdown และทุกอย่างด้านล่างหายไปหมด)

---

### 3.2 Flow: ยืนยันโอนเงินให้พนักงาน + แนบสลิป — **flow หลักและสำคัญที่สุดของหน้านี้**

**ขั้นตอน**
1. เลือกรอบที่สถานะ **"ปิดรอบแล้ว"** (รอบที่มีรายการ payout) → เห็นการ์ดรายคนเรียงยอดมาก→น้อย
2. มองหาคนที่ยังไม่โอน — สังเกตจาก **การ์ดที่มีปุ่มสีหลักด้านขวา และไม่มี badge เขียว**
3. แตะปุ่ม **"ยืนยันโอน + แนบสลิป"** → เปิด modal (bottom sheet บนมือถือ) หัวข้อไดนามิก: `ยืนยันโอนให้ {ชื่อพนักงาน}` ตัวอย่างจริง: **"ยืนยันโอนให้ สมชาย ใจดี"** (ถ้าชื่อว่าง → **"ยืนยันโอนให้ พนักงาน"**)
4. ใน modal อ่านกล่องสรุปยอด: กล่องพื้นเทาอ่อน ขอบมน ในกล่องมี ซ้าย = **"ยอดที่ต้องโอน"** (สีเทา) / ขวา = ยอดเงินตัวหนา เช่น **"฿3,250"**
5. **ออกจากแอปไปโอนเงินจริงผ่านแอปธนาคาร** — ⚠️ ระบบนี้ **ไม่โอนเงินให้** เป็นเพียงการบันทึกว่าโอนแล้ว
6. กลับมาที่ modal → แตะกล่องแนบรูป (กล่องเส้นประ พื้นเทาอ่อน สูงอย่างน้อยประมาณ 96px มีอิโมจิ 📷 ตัวใหญ่ และข้อความ **"แนบสลิปการโอน (บังคับ)"**) → กล้องหลังของมือถือเปิดขึ้นทันที หรือเลือกรูปจากคลังภาพ
   - ใต้กล่องมีข้อความช่วยตัวเล็กสีเทาจาง: **"ถ่ายหรือเลือกรูปสลิปที่โอนให้พนักงานคนนี้"**
   - เหนือกล่องมี label ตัวเล็กตัวหนา: **"สลิปการโอน"**
7. เลือกรูปแล้ว → กล่องเปลี่ยนเป็นรูปพรีวิว (สูงไม่เกินประมาณ 224px ครอปเต็มความกว้าง) มีแถบไล่เฉดดำที่ขอบล่าง และปุ่มขาวโปร่งเล็กมุมขวาล่าง ข้อความ **"ถ่ายใหม่"**
8. (ถ้ารูปไม่ชัด) แตะ **"ถ่ายใหม่"** → เลือก/ถ่ายรูปใหม่ทับของเดิม
9. ปุ่มยืนยันเต็มความกว้างด้านล่าง **"ยืนยันโอนแล้ว"** จะกดได้ก็ต่อเมื่อ **มีรูปแล้วเท่านั้น** → แตะปุ่ม
10. ปุ่มเปลี่ยนข้อความเป็น **"กำลังบันทึก…"** และกดไม่ได้ → ระบบอัปโหลดรูปเข้าที่เก็บส่วนตัว แล้วเรียกเซิร์ฟเวอร์บันทึกการโอน
11. เซิร์ฟเวอร์เช็คว่าเป็น admin จริง เช็คว่ารายการนี้อยู่ในร้านเดียวกัน แล้วบันทึกเวลาโอน + ผู้ยืนยัน + ที่อยู่ไฟล์สลิป — **เฉพาะกรณีที่ยังไม่เคยถูกยืนยันมาก่อน** (กันการจ่ายซ้ำ)
12. เซิร์ฟเวอร์ยิงแจ้งเตือน LINE ไปหาพนักงานคนนั้นอัตโนมัติ
13. สำเร็จ → toast เขียวโผล่ → **modal ปิดเอง** → รายการโหลดใหม่ → การ์ดคนนั้นเปลี่ยนเป็น badge **"✅ โอนแล้ว"** + วันเวลา + ปุ่ม **"ดูสลิปการโอน"**
14. วนทำซ้ำกับคนถัดไป

**กฎการตรวจสอบ (validation)**

| กฎ | ผลเมื่อไม่ผ่าน |
|---|---|
| **ต้องแนบรูปสลิป (บังคับ)** | ปุ่ม "ยืนยันโอนแล้ว" กดไม่ได้ (disabled) ตราบใดที่ยังไม่มีรูป — และมีด่านซ้อนอีกชั้น ถ้าหลุดมาได้จะขึ้น toast แดง **"กรุณาแนบรูปสลิปการโอน"** |
| ต้องมีโปรไฟล์ผู้ใช้ใน session | toast แดง **"ไม่พบโปรไฟล์ผู้ใช้ กรุณาเข้าสู่ระบบใหม่"** |
| ปุ่มถูก disable ระหว่างกำลังบันทึก | ปุ่มเป็นสีจาง กดไม่ได้ ข้อความ "กำลังบันทึก…" |
| ไฟล์รับเฉพาะรูปภาพ | ตัวเลือกไฟล์กรองให้เอง |
| **ไม่มีการจำกัดขนาดไฟล์ และไม่มีการย่อ/บีบอัดรูป** | รูปจากกล้อง/แคปหน้าจอขนาดหลาย MB จะถูกอัปตรงๆ — **ไม่มี progress bar** ผู้ใช้เห็นแค่ "กำลังบันทึก…" |
| ฝั่งเซิร์ฟเวอร์: ต้องเป็น role admin | ปฏิเสธคำขอ |
| ฝั่งเซิร์ฟเวอร์: รายการต้องอยู่ในร้านเดียวกัน | ไม่พบรายการ |
| **ฝั่งเซิร์ฟเวอร์: กันยืนยันซ้ำ** — บันทึกได้เฉพาะรายการที่ยังไม่เคยยืนยัน | toast แดง **"รายการนี้ถูกยืนยันโอนไปแล้ว"** |
| **ไม่มีช่องกรอกจำนวนเงิน ไม่มีช่องหมายเหตุ** | ยอดถูกกำหนดมาจากเซิร์ฟเวอร์เท่านั้น แอดมินแก้ยอดจากหน้านี้ไม่ได้ |
| **ไม่มี dialog ถามยืนยันซ้ำ** | กดปุ่มแล้วบันทึกทันที |

**ข้อความสำเร็จ (verbatim):**
> toast สีเขียว ตัวอักษรขาว: **"บันทึกการโอนแล้ว ระบบแจ้งพนักงานให้เรียบร้อย"**

แสดงค้างประมาณ 3.6 วินาที ลอยเหนือ tab bar (มือถือ) / มุมล่าง (จอใหญ่) จากนั้น modal ปิดเอง การ์ดอัปเดตเป็น "✅ โอนแล้ว"

**ข้อความล้มเหลว (verbatim):** toast สีแดง แสดงข้อความจริงจากเซิร์ฟเวอร์ถ้ามี มิฉะนั้นใช้ข้อความสำรอง
- **"รายการนี้ถูกยืนยันโอนไปแล้ว"** — กรณีมีคนยืนยันคนนี้ไปก่อนแล้ว
- **"บันทึกไม่สำเร็จ"** — ข้อความสำรองทั่วไป
- **"ทำรายการไม่สำเร็จ"** — ข้อความสำรองจากชั้นเรียกเซิร์ฟเวอร์
- กรณีอัปโหลดรูปล้มเหลว → toast แดงแสดงข้อความ error ของระบบไฟล์ตรงๆ (**อาจเป็นภาษาอังกฤษ ไม่ผ่านการแปล**)

**พฤติกรรมหลังล้มเหลว:** ปุ่มกลับมากดได้อีกครั้ง, **modal ยังเปิดค้างพร้อมรูปที่เลือกไว้ (ไม่หาย)** — ลองใหม่ได้ทันที

### 3.3 Flow: ดูสลิปย้อนหลังของรายการที่โอนแล้ว

**ขั้นตอน**
1. ที่การ์ดที่มี badge **"✅ โอนแล้ว"** และมีสลิปแนบ → แตะปุ่มโปร่งเต็มความกว้าง **"ดูสลิปการโอน"**
2. เปิด modal หัวข้อไดนามิก: `สลิปโอนให้ {ชื่อพนักงาน}` เช่น **"สลิปโอนให้ สมชาย ใจดี"**
3. ระหว่างระบบสร้างลิงก์รูปชั่วคราว → วงกลมหมุน + ข้อความ **"กำลังโหลดสลิป…"**
4. สำเร็จ → เห็นรูปสลิปเต็มความกว้าง มุมมน แสดงทั้งรูปไม่ครอป
5. ใต้รูปมีข้อความตัวเล็กสีเทาจางกึ่งกลาง: `โอนเมื่อ {วันเวลา}` ตัวอย่างจริง: **"โอนเมื่อ 15/7/2568 14:30"**
6. ปิดด้วยปุ่ม **"✕"** มุมขวาบน หรือแตะพื้นหลังสีดำโปร่ง

**กฎการตรวจสอบ (validation)**
- ปุ่ม "ดูสลิปการโอน" **แสดงเฉพาะเมื่อโอนแล้ว และมีสลิปแนบจริง** — รายการที่โอนแล้วแต่ไม่มีสลิป (ข้อมูลเก่า) จะไม่มีปุ่มนี้
- ลิงก์รูปเป็นลิงก์ชั่วคราว **อายุ 1 ชั่วโมง** ต้องสร้างใหม่ทุกครั้งที่เปิดดู (จำผลไว้ 30 นาที) → **จึงมี loading state เสมอ แม้เป็นรูปที่เพิ่งอัปไปเมื่อครู่**
- **ไม่มีปุ่มลบ / แก้ไข / เปลี่ยนสลิป / ดาวน์โหลด / ซูม** — ดูอย่างเดียว เมื่อยืนยันโอนแล้วแก้จากหน้านี้ไม่ได้เลย

**ข้อความสำเร็จ:** ไม่มี toast — รูปแสดงใน modal พร้อมบรรทัด "โอนเมื่อ …"

**ข้อความล้มเหลว (verbatim):** ใน modal แทนที่ตำแหน่งรูปด้วย
> ⚠️
> **"โหลดข้อมูลไม่สำเร็จ"**
> **"อาจเป็นเน็ตหลุดหรือเซสชันหมดอายุ — ข้อมูลยังอยู่ในระบบ"**
> ปุ่ม **"ลองใหม่"**

⚠️ **หมายเหตุสำคัญ:** ระบบมีข้อความ error ที่เจาะจงกว่านี้อยู่ภายใน (**"รายการนี้ไม่มีสลิปแนบ"** และ **"สร้างลิงก์รูปสลิปไม่สำเร็จ"**) แต่ **ไม่ถูกแสดงบนจอ** ผู้ใช้เห็นแค่ข้อความกลางๆ เสมอ — นี่คือจุดที่ดีไซน์ใหม่ปรับปรุงได้

---

### 3.4 Flow: ดูยอดสะสมต่อคนของรอบที่ยังไม่ตัด (read-only)

**ขั้นตอน**
1. เลือกรอบที่สถานะ **"กำลังสะสม"** (หรือรอบใดก็ตามที่ยังไม่มีรายการ payout)
2. ระบบรวมยอดค่ารอบ/ค่าคอมของแต่ละคนในรอบนั้นให้อัตโนมัติ
3. เห็นหัวข้อ **"ยอดสะสมต่อคน (ยังไม่ตัดรอบ)"** ตามด้วยการ์ดรายคน เรียงยอดมาก→น้อย: ชื่อซ้าย / ยอดบาทขวา
4. จบ — **ไม่มี action ใดๆ ให้ทำในโหมดนี้** ไม่มีปุ่มตัดรอบ ไม่มีปุ่มสร้าง payout กดการ์ดไม่ได้

**กฎการตรวจสอบ (validation)**
- ไม่มี input จึงไม่มี validation
- ชื่อที่หายไปแสดงเป็น **"พนักงาน"**
- **ถ้ารอบมีทั้ง payout และยอดสะสม → payout ชนะเสมอ** และส่วนยอดสะสมจะไม่แสดงเลย

**ข้อความสำเร็จ:** ไม่มี toast
**ข้อความล้มเหลว:** ErrorState เดิม (⚠️ / "โหลดข้อมูลไม่สำเร็จ" / "อาจเป็นเน็ตหลุดหรือเซสชันหมดอายุ — ข้อมูลยังอยู่ในระบบ" / ปุ่ม "ลองใหม่") — **แสดงได้ทีละอันเท่านั้น** ถ้าทั้ง payout และ items ล้มเหลวพร้อมกัน จะเห็นแค่อันแรก

---

### 3.5 Flow: ผู้ไม่มีสิทธิ์เข้าหน้านี้ (permission-denied)

**ขั้นตอน**
1. พนักงาน role อื่น (stock / packer / rider / staff) **ไม่เห็นเมนู "💰 เงินเดือน"** ใน tab bar ตั้งแต่แรก
2. ถ้าพิมพ์ที่อยู่หน้านี้เข้ามาตรงๆ → **ถูกเด้งไปหน้าแรกของ role ตัวเองทันที**
3. **ไม่มีข้อความ "คุณไม่มีสิทธิ์" ไม่มี toast ไม่มีหน้าจอกลาง** — เปลี่ยนหน้าเงียบๆ

**กฎการตรวจสอบ:** ต้องมี role `admin` เท่านั้น (เช็คทั้งฝั่งหน้าเว็บและฝั่งเซิร์ฟเวอร์)
**ข้อความสำเร็จ/ล้มเหลว:** ไม่มีข้อความใดๆ — ⚠️ นี่คือช่องว่างของดีไซน์ปัจจุบัน (ดู §8)

## 4. สถานะของหน้า

| สถานะ | ผู้ใช้เห็นอะไร | ข้อความจริง |
|---|---|---|
| **ปกติ (รอบตัดแล้ว)** | หัวข้อ + dropdown + การ์ดสรุป (badge เขียว) + การ์ด payout รายคน เรียงยอดมาก→น้อย ผสมกันระหว่างคนที่มีปุ่มกับคนที่มี badge เขียว | "ปิดรอบแล้ว" / "รวม ฿12,500" / "ยืนยันโอน + แนบสลิป" / "✅ โอนแล้ว" / "ดูสลิปการโอน" |
| **ปกติ (รอบยังไม่ตัด)** | หัวข้อ + dropdown + การ์ดสรุป (badge เหลือง) + หัวข้อกลุ่ม + การ์ดยอดสะสม read-only | "กำลังสะสม" / "ยอดสะสมต่อคน (ยังไม่ตัดรอบ)" |
| **กำลังโหลด — ทั้งหน้า** | หัวข้อค้างอยู่ ทุกอย่างใต้หัวข้อถูกแทนด้วยวงกลมหมุน + ข้อความสีเทา จัดกึ่งกลาง มีระยะห่างบน-ล่างพอควร (**ไม่มี skeleton loader ที่ใดในหน้านี้เลย**) | "กำลังโหลด…" |
| **กำลังโหลด — เฉพาะรายการ (ตอนสลับรอบ)** | dropdown + การ์ดสรุปรอบยังคาอยู่ วงกลมหมุนแทนเฉพาะพื้นที่รายการด้านล่าง | "กำลังโหลด…" |
| **กำลังโหลด — สลิปใน modal** | วงกลมหมุน + ข้อความ ในกล่อง modal | "กำลังโหลดสลิป…" |
| **ว่าง — ไม่มีรอบเลย** | อิโมจิใหญ่ 💰 + หัวข้อ + hint จัดกึ่งกลาง — **ซ่อน dropdown และการ์ดสรุปทั้งหมด** | 💰 / "ยังไม่มีรอบจ่ายเงิน" / "รอบจะสร้างอัตโนมัติเมื่อมีค่ารอบ/ค่าคอม" |
| **ว่าง — มีรอบแต่รอบนี้ไม่มีรายการ** | อิโมจิใหญ่ 🧾 + หัวข้อ + hint — **dropdown และการ์ดสรุปยังแสดง** โดยการ์ดสรุปโชว์ "รวม ฿0" | 🧾 / "ยังไม่มีรายการในรอบนี้" / "ค่ารอบ/ค่าคอมจะปรากฏเมื่อมีงานเสร็จในรอบนี้" |
| **ผิดพลาด (ทุกจุด ใช้หน้าตาเดียวกันหมด)** | อิโมจิ ⚠️ ตัวใหญ่ + หัวข้อตัวหนา + hint ตัวเล็กสีจาง + ปุ่มโปร่งขนาดเล็ก | ⚠️ / "โหลดข้อมูลไม่สำเร็จ" / "อาจเป็นเน็ตหลุดหรือเซสชันหมดอายุ — ข้อมูลยังอยู่ในระบบ" / "ลองใหม่" |
| **กำลังส่ง (busy) — ใน modal ยืนยันโอน** | ปุ่มหลักเป็นสีจาง กดไม่ได้ ข้อความเปลี่ยน. **ไม่มี overlay คลุม modal, ไม่มี progress bar, และปุ่ม ✕ กับการแตะพื้นหลังยังปิด modal ได้ระหว่างกำลังบันทึก** | "กำลังบันทึก…" (ปกติคือ "ยืนยันโอนแล้ว") |
| **กำลังส่ง — ปุ่มยังกดไม่ได้ (ยังไม่แนบรูป)** | ปุ่มหลักสีจาง กดไม่ได้ ตั้งแต่เปิด modal | "ยืนยันโอนแล้ว" (สีจาง) |
| **ไม่มีสิทธิ์ (permission-denied)** | **ไม่เห็นหน้านี้เลย** — เมนูไม่โผล่ และถ้าเข้าตรงๆ จะถูกเด้งออกเงียบๆ | (ไม่มีข้อความใดๆ) |

## 7. ข้อมูลอัปเดตยังไง

**ไม่มี realtime และไม่มี polling ในหน้านี้เลย** — ทุกอย่างเป็นการโหลดตามเหตุการณ์เท่านั้น

ข้อมูลจะโหลดใหม่ก็ต่อเมื่อ:
1. เปิดหน้าครั้งแรก
2. สลับรอบใน dropdown → รายการของรอบใหม่ถูกโหลด
3. **ยืนยันโอนสำเร็จ** → รายการ payout ถูกโหลดใหม่ทันทีอัตโนมัติ
4. กดปุ่ม "ลองใหม่" ใน error state
5. กลับมาโฟกัสที่หน้าต่างเบราว์เซอร์อีกครั้ง (พฤติกรรมมาตรฐานของ data layer)

**ลิงก์รูปสลิป** ถูกจำไว้ 30 นาที แต่ตัวลิงก์เองหมดอายุใน 1 ชั่วโมง → เปิด modal ดูสลิปครั้งแรกจะเห็น "กำลังโหลดสลิป…" เสมอ

### ผลต่อดีไซน์ — สำคัญ
- **หน้าจอนี้ค้างเก่าได้ (stale)** ถ้าแอดมินคนที่สองยืนยันโอนคนเดียวกันจากอีกเครื่อง หน้าจอของเราจะยัง **โชว์ปุ่ม "ยืนยันโอน + แนบสลิป" ต่อไปเหมือนไม่มีอะไรเกิดขึ้น** จนกว่าจะโหลดใหม่ — ผู้ใช้จะรู้ตอนกดปุ่มแล้วโดน toast แดง "รายการนี้ถูกยืนยันโอนไปแล้ว" เท่านั้น (และรูปที่เขาอัปไปแล้วกลายเป็นไฟล์กำพร้าในระบบ)
- **ปัจจุบันไม่มีตัวบอกว่าอัปเดตล่าสุดเมื่อไหร่ และไม่มีปุ่มรีเฟรชด้วยตัวเอง** — ดีไซน์ใหม่ **ควรพิจารณาเพิ่ม** ตัวบอกเวลาอัปเดตล่าสุด + ปุ่มรีเฟรช (หรือ pull-to-refresh บนมือถือ) เพราะหน้านี้เกี่ยวกับเงินและมีโอกาสมีแอดมินหลายคนทำพร้อมกัน
- ในทางกลับกัน **ไม่ควรใส่ polling ถี่ๆ** ให้ค่าใช้จ่ายสูงโดยไม่จำเป็น — หน้านี้ใช้สัปดาห์ละครั้ง ปุ่มรีเฟรชแบบชัดเจนน่าจะเหมาะกว่า

## 8. สิ่งที่ designer ต้องรู้

### 🔒 ห้ามแตะ — ผูกกับกติกาธุรกิจ (Ready.md §3.7) และความปลอดภัยของเงิน

- **การแนบรูปสลิปเป็นเงื่อนไขบังคับ ห้ามทำให้ข้ามได้** — ไม่ว่าจะออกแบบใหม่ยังไง ปุ่มยืนยันต้องกดไม่ได้จนกว่าจะมีรูป นี่คือหลักฐานการโอนตามกติกาธุรกิจ
- **ระบบนี้ไม่โอนเงิน** — แอดมินต้องออกไปโอนผ่านแอปธนาคารเอง หน้านี้แค่ "บันทึกว่าโอนแล้ว" **ห้ามออกแบบให้ดูเหมือนปุ่มโอนเงิน** (เช่น อย่าใช้คำว่า "โอนเลย" หรือไอคอนกดโอน) — คำ verbatim ปัจจุบัน "ยืนยันโอน**แล้ว**" สื่อ past tense ถูกต้องแล้ว
- **ยอดเงินแก้ไม่ได้จากหน้านี้** — ไม่มีช่องกรอกจำนวน ไม่มีหมายเหตุ ห้ามเพิ่ม field เหล่านี้เข้าไปเอง ยอดมาจากระบบหลังบ้านเท่านั้น
- **หน้านี้ไม่มีการสร้างรอบ / ตัดรอบ / ปิดรอบ / export / พิมพ์** — รอบเกิดขึ้นอัตโนมัติจากหลังบ้าน **ห้ามออกแบบปุ่มเหล่านี้ขึ้นมาเอง** (empty state ก็บอกไว้แล้วว่า "รอบจะสร้างอัตโนมัติเมื่อมีค่ารอบ/ค่าคอม")
- **การแจ้งเตือนพนักงานทาง LINE เกิดอัตโนมัติฝั่งเซิร์ฟเวอร์** — แอดมินไม่ต้องกดอะไรเพิ่ม ห้ามเพิ่ม checkbox "แจ้งพนักงานด้วย" toast ก็บอกไว้แล้วว่า "ระบบแจ้งพนักงานให้เรียบร้อย"
- **ห้ามเพิ่มปุ่มยกเลิก / undo / แก้สลิป** — ระบบไม่รองรับ ถ้าออกแบบไปจะเป็น dead UI
- **ข้อความไทยทุกคำใน §5 คือ copy จริง — ห้ามแต่งใหม่** ถ้าคิดว่าคำไหนควรเปลี่ยน ให้เสนอเป็นข้อเสนอแยก อย่าเปลี่ยนเงียบๆ ในไฟล์ดีไซน์

### ✅ ปรับได้ — จุดที่ดีไซน์ปัจจุบันอ่อนและควรปรับปรุง

- **น้ำหนักภาพยังเบาเกินสำหรับ action ที่ irreversible** — นี่คือหน้าที่ผลกระทบสูงที่สุดหน้าหนึ่งของทั้งระบบ (จ่ายเงินจริง แก้ไม่ได้) แต่ปัจจุบัน: ไม่มี confirm ซ้อน ไม่มีการย้ำยอด ไม่มีสีเตือน ปุ่มก็เป็นปุ่มหลักธรรมดาเหมือนปุ่มอื่นๆ ทั้งแอป **ควรพิจารณาให้ modal ยืนยันสื่อความ "กดแล้วย้อนไม่ได้"**
- **"ยังไม่โอน" ไม่มี badge เลย** — สื่อด้วยการมีปุ่มเท่านั้น ขณะที่ "โอนแล้ว" มี badge เขียว → **สแกนรายการยาวๆ ว่าใครยังไม่ได้เงินทำได้แย่มาก** ควรมีสัญญาณสถานะที่สมมาตรกันทั้งสองฝั่ง
- **ไม่มีตัวสรุปความคืบหน้าใดๆ** — ไม่มี "โอนแล้ว 5/12 คน" ไม่มี "ยอดคงเหลือที่ต้องโอน ฿X" การ์ดสรุปโชว์แค่ยอดรวมทั้งรอบ ทั้งที่นี่คือข้อมูลที่แอดมินอยากรู้ที่สุดระหว่างไล่จ่าย
- **ไม่มี filter / search / pagination ในรายการคน** — ทีมใหญ่ = list ยาวมากบนมือถือ ต้องเลื่อนหาเอง อย่างน้อยควรมีตัวกรอง "ยังไม่โอน"
- **สองโหมดหน้าตาคล้ายกันแต่ทำอะไรได้ต่างกันมาก** — โหมด "ยอดสะสม" (read-only) กับโหมด "payout" (จ่ายได้) ใช้โครงการ์ดเหมือนกัน (ชื่อซ้าย/ยอดขวา) แถมโหมดสะสมมีหัวข้อกำกับ แต่โหมด payout **ไม่มีหัวข้อเลย** → เสี่ยงสับสน ควรแยกให้ต่างชัดกว่านี้
- **flow ข้ามแอปเปราะบาง** — แอดมินต้องออกไปแอปธนาคารแล้วกลับมา แต่ **modal ปิดหายถ้าเผลอแตะพื้นหลัง และรูปที่เลือกไว้หายไปด้วย** ควรออกแบบให้กลับมาแล้วหาที่ค้างไว้ได้ง่าย และ/หรือกันการปิดโดยไม่ตั้งใจ
- **ระหว่างอัปโหลดรูปไม่มี progress เลย** — รูปไม่ถูกย่อ/บีบอัด (ต่างจากรูปสินค้าที่ระบบย่อให้) แคปหน้าจอสลิปหลาย MB บนเน็ตช้าจะรู้สึกเหมือนแอปค้าง โดยเห็นแค่คำว่า "กำลังบันทึก…" — **ต้องการสัญญาณ progress**
- **ระหว่างกำลังบันทึก ยังปิด modal ได้** (ปุ่ม ✕ และการแตะพื้นหลังยังทำงาน) → กดปิดกลางคันได้ทั้งที่ระบบยังบันทึกอยู่ ควรกันไว้
- **คำว่า "ถ่ายใหม่" อาจไม่ตรงพฤติกรรมจริง** — ระบบเปิดกล้องหลังให้ทันที แต่กรณีจริงส่วนใหญ่แอดมินจะ "เลือกรูปจากคลัง" (แคปหน้าจอสลิปธนาคาร) มากกว่าถ่ายใหม่
- **ข้อความ error ที่เจาะจงถูกกลืนหาย** — "รายการนี้ไม่มีสลิปแนบ" / "สร้างลิงก์รูปสลิปไม่สำเร็จ" มีอยู่ในระบบแต่ไม่แสดงบนจอ ผู้ใช้เห็นแต่ "โหลดข้อมูลไม่สำเร็จ" กลางๆ เสมอ
- **error ของการอัปโหลดรูปอาจเป็นภาษาอังกฤษดิบ** — โผล่ใน toast แดงตรงๆ ไม่ผ่านการแปล
- **ไม่มีปุ่มรีเฟรช ไม่มีตัวบอกเวลาอัปเดตล่าสุด** — ดู §7
- **จอใหญ่โล่งมาก** — คอลัมน์เดียวล้วน ไม่มี grid ทั้งที่พื้นที่กว้าง มีที่ให้เล่นเยอะ (เช่น 2 คอลัมน์ หรือ layout แบบตาราง)
- **ไม่มี skeleton loader ที่ใดเลย** — มีแต่วงกลมหมุน
- **ความไม่สอดคล้องเล็กๆ ที่ควรเก็บกวาด:** ตัวคั่นช่วงวันที่ใน dropdown เป็น en-dash `–` แต่การ์ดสรุปเป็นลูกศร `→`; ฟอร์แมตวันที่ 2 แบบปนกันในหน้าเดียว (`15 ก.ค. 68` กับ `15/7/2568 14:30`); คำสถานะรอบซ้ำกัน 2 ที่ (ใน badge และต่อท้าย option)
- **26 รอบล่าสุดคือเพดานตายตัว** — ไม่มีทางเข้าถึงรอบที่เก่ากว่านั้นจากหน้านี้ ถ้าดีไซน์ใหม่จะมี date picker หรือช่องค้นหารอบ ต้องคุยกับ engineer ก่อน (เป็นข้อจำกัดของข้อมูล ไม่ใช่ของ UI)
- **ไม่มีทางดูรายละเอียดว่ายอดของแต่ละคนมาจากงานอะไรบ้าง** — กดชื่อคนไม่ได้ เห็นแค่ยอดรวม ถ้าดีไซน์ใหม่จะเพิ่ม drill-down ต้องคุยกับ engineer ก่อน

### ♿ Accessibility ที่มีอยู่แล้วและ **ควรรักษาไว้**

- ปุ่มปิด modal มีป้ายช่วยการเข้าถึงว่า "ปิด"
- รูปทุกใบมีคำอธิบายภาษาไทย (`ตัวอย่างรูป` / `สลิปการโอนค่าจ้าง`)
- กล่องแนบรูปแตะได้ทั้งกล่อง (ไม่ใช่แค่ไอคอน) และสูงอย่างน้อยประมาณ 96px
- tab bar สูงอย่างน้อยประมาณ 56px
- **ระยะเผื่อท้ายหน้า:** เนื้อหาต้องเผื่อพื้นที่ด้านล่างไม่ให้ tab bar (มือถือ) ทับ และ toast ลอยอยู่เหนือ tab bar เสมอ
